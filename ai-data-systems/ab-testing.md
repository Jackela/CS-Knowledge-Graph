# A/B测试 (A/B Testing)

## 概念

A/B测试（A/B Testing）是一种统计方法，用于比较两个版本的产品或功能，以确定哪个版本表现更好。通过将用户随机分配到不同组，对比关键指标来做出数据驱动的决策。

## 核心原理

```
┌─────────────────────────────────────────┐
│              A/B测试流程                 │
├─────────────────────────────────────────┤
│                                         │
│   1. 假设提出 → 2. 分组实验              │
│        ↓              ↓                 │
│   3. 数据收集 → 4. 统计检验              │
│        ↓              ↓                 │
│   5. 决策制定                           │
│                                         │
└─────────────────────────────────────────┘
```

## 应用场景

- **产品功能**: 新功能是否提升用户参与度
- **界面设计**: 哪个UI版本转化率更高
- **推荐算法**: 哪种推荐策略效果更好
- **定价策略**: 不同价格点的收益比较

## 关键指标

| 指标 | 说明 | 适用场景 |
|------|------|----------|
| 转化率 | 用户完成目标行为的比例 | 电商、注册 |
| 点击率 | 点击次数/展示次数 | 广告、推荐 |
| 留存率 | 用户持续使用的比例 | 产品功能 |
| 收益 | 收入或利润变化 | 定价、变现 |

## 统计基础

### 样本量计算

```
n = 16 * σ² / δ²

其中:
- σ: 标准差
- δ: 期望检测的最小差异
- n: 每组所需样本量
```

### 显著性检验

- **p值**: 小于0.05通常认为显著
- **置信区间**: 95%置信区间不包含0表示显著
- **统计功效**: 通常要求80%以上

## 实施步骤

1. **明确目标**: 定义要优化的核心指标
2. **形成假设**: 提出可验证的改进假设
3. **设计实验**: 确定分组、样本量、持续时间
4. **实施测试**: 随机分流，确保实验组和对照组独立
5. **收集数据**: 追踪关键指标和用户行为
6. **分析结果**: 统计检验，得出结论
7. **决策行动**: 根据结果决定是否全量发布

## 注意事项

- **避免过早停止**: 样本不足可能导致错误结论
- **控制变量**: 只改变一个因素，保持其他条件一致
- **长期影响**: 短期效果可能与长期效果不同
- **伦理考虑**: 确保实验不会对用户造成伤害
