# RAG (Retrieval-Augmented Generation，检索增强生成)

## 简介

**RAG (Retrieval-Augmented Generation)** 是一种将信息检索技术与文本生成模型相结合的AI架构。它通过在大语言模型生成答案之前，先从外部知识库中检索相关信息，从而显著提升回答的准确性、时效性和可信度。

**RAG的优势：**
- **减少幻觉**：生成内容基于检索到的真实文档
- **知识更新**：无需重新训练模型即可更新知识
- **可追溯性**：答案可追溯到具体来源文档
- **成本效益**：比微调大模型更经济
- **数据安全**：私有数据无需上传到模型提供商

### 1. 索引阶段 (Indexing)

```
文档处理流水线：

原始文档
    │
    ▼
┌─────────────┐
│  文档加载   │  ← PDF、Word、网页、数据库等
│  (Loading)  │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  文档分块   │  ← 按段落、句子或固定长度切分
│  (Chunking) │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  文本清洗   │  ← 去除噪声、格式化
│ (Cleaning)  │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  向量化     │  ← Embedding模型编码为向量
│(Embedding)  │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  索引存储   │  ← 存入向量数据库
│  (Storage)  │
└─────────────┘
```

**文档分块策略：**

| 策略 | 描述 | 适用场景 |
|------|------|----------|
| 固定长度 | 按固定字符/Token数切分 | 通用场景 |
| 递归切分 | 优先按段落、句子层级切分 | 保持语义完整 |
| 语义切分 | 基于语义相似度确定边界 | 高精度需求 |
| 结构切分 | 按文档结构（标题、章节） | 结构化文档 |

## 检索方法

### 1. 稠密检索 (Dense Retrieval)

```
稠密检索原理：

文本 ──▶ Embedding模型 ──▶ 稠密向量 (Dense Vector)
                              ↓
                    ┌──────────────────┐
                    │  768维向量空间    │
                    │                  │
                    │    ● 文档A       │
                    │      ↘           │
                    │    ●───● 查询Q   │  ← 余弦相似度计算
                    │      ↗           │
                    │    ● 文档B       │
                    │                  │
                    └──────────────────┘

特点：
- 使用神经网络模型（如BERT、Sentence-BERT）生成向量
- 捕获语义信息，支持语义相似匹配
- 向量维度通常较高（768/1024/1536维）
- 需要向量数据库支持ANN检索
```

**常用Embedding模型：**

| 模型 | 维度 | 语言 | 特点 |
|------|------|------|------|
| text-embedding-3-large | 3072 | 多语言 | OpenAI，效果优秀 |
| text-embedding-ada-002 | 1536 | 多语言 | OpenAI，性价比高 |
| BGE-Large | 1024 | 中英 | 开源，中文表现好 |
| GTE-Large | 1024 | 多语言 | 阿里开源 |
| E5-Mistral | 4096 | 多语言 | 微软开源 |
| m3e-base | 768 | 中英 | 开源轻量 |

### 2. 稀疏检索 (Sparse Retrieval)

```
稀疏检索原理（BM25）：

文档集：
D1: "机器学习是人工智能的一个分支"
D2: "深度学习是机器学习的子领域"
D3: "人工智能改变未来"

倒排索引：
┌────────┬──────────────────────────┐
│  词项   │ 文档列表 (TF-IDF/BM25)   │
├────────┼──────────────────────────┤
│ 人工智能 │  D1(0.5), D3(0.8)        │
│ 机器学习 │  D1(0.6), D2(0.7)        │
│ 深度学习 │  D2(0.9)                 │
│ ...    │  ...                     │
└────────┴──────────────────────────┘

查询: "人工智能与机器学习"
    ↓
评分计算：
Score(D1) = BM25("人工智能", D1) + BM25("机器学习", D1) = 1.1
Score(D2) = BM25("机器学习", D2) = 0.7
Score(D3) = BM25("人工智能", D3) = 0.8
    ↓
排序结果：D1 > D3 > D2

特点：
- 基于词项匹配（如TF-IDF、BM25）
- 可解释性强，精确匹配关键词
- 不需要神经网络模型
- 适合关键词搜索场景
```

**BM25公式：**

```
BM25(D, Q) = Σ IDF(qᵢ) · [f(qᵢ, D) · (k₁ + 1)] / [f(qᵢ, D) + k₁ · (1 - b + b · |D|/avgdl)]

其中：
- f(qᵢ, D): 词项qᵢ在文档D中的频率
- |D|: 文档长度
- avgdl: 平均文档长度
- k₁, b: 调节参数
```

### 3. 混合检索 (Hybrid Retrieval)

```
混合检索架构：

用户查询
    │
    ├──▶ 稠密检索 ──┐
    │   (向量相似度)  │
    │                 ├──▶ 结果融合 ──▶ 重排序 ──▶ 最终结果
    │                 │    (RRF/CC)      (Reranker)
    └──▶ 稀疏检索 ──┘
        (关键词匹配)

融合方法：

1. RRF (Reciprocal Rank Fusion)
   score = Σ 1/(k + rankᵢ)
   
   例：
   稠密检索排名: D1(1), D3(2), D2(3)
   稀疏检索排名: D2(1), D1(2), D3(3)
   
   RRF分数 (k=60):
   D1: 1/61 + 1/62 = 0.0325
   D2: 1/63 + 1/61 = 0.0323
   D3: 1/62 + 1/63 = 0.0319
   
   最终排序: D1 > D2 > D3

2. 线性加权
   score = α · score_dense + (1-α) · score_sparse
```

**重排序 (Reranking)：**

```
初排结果 (Top-100) ──▶ 重排序模型 ──▶ 精排结果 (Top-10)

重排序模型：
- Cross-Encoder: 将查询和文档拼接后输入BERT，获得更精确的相似度
- Cohere Rerank: 商业API
- bge-reranker: 开源重排序模型

Cross-Encoder优势：
- 能捕捉查询和文档的细粒度交互
- 精度高于双塔模型
- 计算量大，只对Top-K重排
```


**向量数据库核心能力：**

| 功能 | 说明 |
|------|------|
| ANN检索 | 近似最近邻搜索，毫秒级返回相似向量 |
| 向量索引 | HNSW、IVF等索引结构加速检索 |
| 混合查询 | 向量相似度 + 元数据过滤 |
| 多租户 | 支持Collection/Partition隔离 |
| 持久化 | 数据持久化存储和高可用 |

**RAG中的向量索引选择：**

```
索引类型对比：

HNSW (Hierarchical Navigable Small World):
┌──────────────────────────────────────────┐
│  构建图结构，多层导航                      │
│  查询复杂度: O(log N)                     │
│  召回率: 高 (0.95+)                       │
│  内存占用: 高                             │
│  适用: 高精度、内存充足场景                │
└──────────────────────────────────────────┘

IVF (Inverted File Index):
┌──────────────────────────────────────────┐
│  K-means聚类，缩小搜索空间                │
│  查询复杂度: O(√N) ~ O(N)                 │
│  召回率: 中等 (0.85-0.95)                 │
│  内存占用: 低                             │
│  适用: 大规模数据、内存受限                │
└──────────────────────────────────────────┘

FLAT (暴力搜索):
┌──────────────────────────────────────────┐
│  全量计算，精确结果                        │
│  查询复杂度: O(N)                         │
│  召回率: 100%                             │
│  适用: 小规模数据集(<10万)                 │
└──────────────────────────────────────────┘
```

## RAG优化技术

### 1. 查询优化

```
查询重写 (Query Rewriting)：

原始查询: "这个多少钱？"
    ↓
查询扩展: "这个产品的价格是多少？"
    ↓
HyDE (Hypothetical Document Embedding):
   生成假设答案 ──▶ 用假设答案检索相似文档
```

### 2. 多路召回

```
┌─────────────────────────────────────────┐
│           多路召回架构                   │
├─────────────────────────────────────────┤
│                                         │
│  查询 ──┬──▶ 向量检索 ──┐               │
│       ├──▶ 关键词检索 ──┼──▶ 融合排序    │
│       ├──▶ 图谱检索 ────┤               │
│       └──▶ 结构化查询 ──┘               │
│                                         │
└─────────────────────────────────────────┘
```

### 3. 迭代检索

```
迭代RAG (Self-RAG / Corrective RAG)：

初始查询 ──▶ 检索 ──▶ 评估结果质量 ──┬─▶ 生成答案
                                    │
                                    └─▶ 改进查询 ──▶ 重新检索
```

## 面试要点

### 常见问题

**Q1: RAG和微调(Fine-tuning)的区别？**
> RAG通过检索外部知识来增强生成，无需修改模型参数，知识更新只需更新文档库；微调需要重新训练模型参数，适合改变模型行为或风格，但知识更新成本高。两者可以结合使用。

**Q2: 如何解决RAG中的上下文长度限制问题？**
> 1) 使用更长的上下文窗口模型（如Claude 100K+）2) 采用Map-Reduce策略分块处理后汇总 3) 使用压缩技术提取关键信息 4) 迭代筛选最相关的文档片段。

**Q3: 稠密检索和稀疏检索各有什么优缺点？**
> 稠密检索捕获语义信息，适合概念匹配但计算资源要求高；稀疏检索基于词项匹配，精确但无法理解语义。实际应用中常采用混合检索结合两者优势。

**Q4: 如何评估RAG系统的效果？**
> 检索阶段：命中率(Hit Rate)、MRR、NDCG；生成阶段：ROUGE、BLEU、BERTScore；端到端：人工评估答案准确性、完整性和相关性。

**Q5: 文档分块大小如何选择？**
> 太小会丢失上下文语义，太大会降低检索精度。通常128-512 tokens，需要根据具体场景和Embedding模型特性实验确定。语义切分通常比固定长度效果更好。
