# 网络层 (Network Layer)

## 简介

**网络层（Network Layer）**是OSI模型的第3层，负责将数据包从源主机传输到目的主机，提供逻辑寻址和路由选择功能。

## 核心功能

- **逻辑寻址**：IP地址标识主机
- **路由选择**：确定数据包传输路径
- **分组转发**：将数据包从输入链路转发到输出链路

## IP协议

### IPv4

```
IPv4头部 (20-60字节):
┌─────┬─────┬─────────────┬─────────────────────────┐
│版本 │ 头长 │ 服务类型(TOS)│        总长度           │ 4字节
├─────┴─────┴─────────────┴─────────────────────────┤
│           标识            │标志│    分片偏移         │ 8字节
├─────────────────────────┴────┴────────────────────┤
│   生存时间(TTL)   │    协议     │     头部校验和      │ 12字节
├───────────────────────────────────────────────────┤
│                   源IP地址                         │ 16字节
├───────────────────────────────────────────────────┤
│                   目的IP地址                       │ 20字节
├───────────────────────────────────────────────────┤
│                   选项 (可选)                       │
└───────────────────────────────────────────────────┘

版本: 4 (IPv4)
头长: 5-15 (单位4字节，即20-60字节)
TTL: 每经过路由器减1，为0时丢弃
协议: 6=TCP, 17=UDP, 1=ICMP
```

### IP地址分类

```
A类: 1.0.0.0 ~ 126.255.255.255    (网络.主机.主机.主机)
B类: 128.0.0.0 ~ 191.255.255.255  (网络.网络.主机.主机)
C类: 192.0.0.0 ~ 223.255.255.255  (网络.网络.网络.主机)
D类: 224.0.0.0 ~ 239.255.255.255  (组播)
E类: 240.0.0.0 ~ 255.255.255.255  (保留)

私有地址:
A类: 10.0.0.0/8
B类: 172.16.0.0/12
C类: 192.168.0.0/16
```

### CIDR（无类别域间路由）

```
IP/前缀长度

192.168.1.0/24 表示:
- 网络地址: 192.168.1.0
- 子网掩码: 255.255.255.0
- 可用主机: 254 (256-2)

/25 划分:
192.168.1.0/25   (0-127)
192.168.1.128/25 (128-255)
```

### IPv6

```
IPv6地址: 128位，8组16进制

2001:0db8:85a3:0000:0000:8a2e:0370:7334
可简写: 2001:db8:85a3::8a2e:370:7334

:: 只能出现一次，替代连续0组
```

**IPv6特性**:
- 更大地址空间 ($2^{128}$)
- 简化头部
- 内置安全性（IPsec）
- 自动配置

## 路由算法

### 距离向量（Distance Vector）

**RIP协议**（Routing Information Protocol）:

```
原理: 每个路由器维护到所有目的地的距离（跳数）

路由器A的路由表:
目的地 │ 下一跳 │ 距离
───────┼───────┼─────
Net1   │  直接 │  1
Net2   │   B   │  2
Net3   │   C   │  3

更新: 周期交换路由表，使用Bellman-Ford算法
最大跳数: 15（16表示不可达）
```

### 链路状态（Link State）

**OSPF协议**（Open Shortest Path First）:

```
原理:
1. 发现邻居（Hello包）
2. 测量链路成本
3. 构建链路状态包（LSA）
4. 泛洪LSA到全网
5. 使用Dijkstra算法计算最短路径

优点:
- 收敛快
- 无计数到无穷大问题
- 支持分层（区域）
```

### 路径向量（Path Vector）

**BGP协议**（Border Gateway Protocol）:

```
原理: 携带完整路径信息，防止环路

用于: 自治系统（AS）间的路由

路径属性:
- AS_PATH: 经过的AS序列
- NEXT_HOP: 下一跳
- LOCAL_PREF: 本地优先级
- MED: 多出口区分
```

## 路由协议对比

| 特性 | RIP | OSPF | BGP |
|------|-----|------|-----|
| 类型 | 距离向量 | 链路状态 | 路径向量 |
| 范围 | 内部网关(IGP) | 内部网关 | 外部网关(EGP) |
| 度量 | 跳数 | 链路成本 | 路径属性 |
| 收敛 | 慢 | 快 | 慢 |
| 算法 | Bellman-Ford | Dijkstra | 路径向量 |

## ARP协议

**地址解析协议（Address Resolution Protocol）**：IP地址 → MAC地址

```
ARP请求:
"谁的IP是192.168.1.2？请告诉我你的MAC地址"

ARP响应:
"我的IP是192.168.1.2，MAC是00:11:22:33:44:55"

ARP缓存: IP-MAC映射表，定期更新
```

## ICMP协议

**Internet Control Message Protocol**：网络诊断和错误报告

```
常见ICMP消息:
- Echo Request/Reply (Ping)
- Destination Unreachable
- Time Exceeded (TTL=0)
- Redirect

Ping命令:
ping 8.8.8.8
发送ICMP Echo Request，等待Reply
```

## NAT（网络地址转换）

```
私网            NAT路由器           公网
192.168.1.2 ──→│转换表│─────────→ 203.0.113.1
               │192.168.1.2:12345│
               │→ 203.0.113.1:50001│

类型:
- 静态NAT: 一对一固定映射
- 动态NAT: 从地址池动态分配
- NAPT/PAT: 端口地址转换（最常用）
```

## 面试要点

### Q1: 子网划分

```
问题: 将192.168.1.0/24划分为4个子网

解: 借2位主机位，/26

子网1: 192.168.1.0/26   (0-63)
子网2: 192.168.1.64/26  (64-127)
子网3: 192.168.1.128/26 (128-191)
子网4: 192.168.1.192/26 (192-255)

每个子网: 62个可用主机
```

### Q2: 路由环路解决

- 距离向量：设置最大跳数、水平分割、毒性逆转
- 链路状态：完整拓扑信息，Dijkstra算法
- 路径向量：携带完整路径，检测环路
