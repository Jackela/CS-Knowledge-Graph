# 关系模型 (Relational Model)

## 简介

**关系模型（Relational Model）**由 Edgar F. Codd 于 1970 年提出，是数据库管理系统的理论基础。它使用**关系（表）**来表示数据及其之间的联系。

## 核心概念

### 关系（Relation）

```
关系 = 表
┌────────┬────────┬────────┐
│  列1   │  列2   │  列3   │
├────────┼────────┼────────┤
│  值    │  值    │  值    │ ← 行1 (元组)
├────────┼────────┼────────┤
│  值    │  值    │  值    │ ← 行2
└────────┴────────┴────────┘

术语对应:
关系    = 表
元组    = 行/记录
属性    = 列/字段
域      = 属性的取值范围
模式    = 表结构定义
```

### 键（Key）

```
学生表:
┌─────────┬────────┬─────────┬────────┐
│ 学号(PK) │  姓名   │ 身份证(UK) │ 班级(FK) │
├─────────┼────────┼─────────┼────────┤
│  001    │  张三   │ 110101.. │   01   │
│  002    │  李四   │ 310101.. │   02   │
└─────────┴────────┴─────────┴────────┘

主键(PK): 唯一标识每行，非空且唯一
外键(FK): 引用其他表的主键
候选键: 可作为主键的属性集
超键: 能唯一标识元组的属性集
```

## 关系代数

### 基本操作

#### 选择（Selection）

```
σ 条件(关系)

选择年龄>20的学生:
σ 年龄>20(学生) 

结果: 年龄大于20的所有行
```

#### 投影（Projection）

```
π 属性列表(关系)

投影姓名和年龄:
π 姓名, 年龄(学生)

结果: 只包含姓名和年龄两列
```

#### 并、交、差

```
R ∪ S: 在R或S中的元组
R ∩ S: 同时在R和S中的元组
R - S: 在R但不在S中的元组

要求: R和S必须相容（相同属性）
```

#### 连接（Join）

```
⋈: 自然连接（按同名属性连接）
⋈θ: θ连接（按条件连接）

学生 ⋈ 班级

学生表:          班级表:
┌─────┬──────┐  ┌─────┬────────┐
│ 学号 │ 班级ID│  │班级ID│ 班级名  │
├─────┼──────┤  ├─────┼────────┤
│ 001 │  01  │  │ 01  │  一班  │
│ 002 │  02  │  │ 02  │  二班  │
└─────┴──────┘  └─────┴────────┘

连接结果:
┌─────┬──────┬────────┐
│ 学号 │ 班级ID│ 班级名  │
├─────┼──────┼────────┤
│ 001 │  01  │  一班  │
│ 002 │  02  │  二班  │
└─────┴──────┴────────┘
```

### 关系代数表达式

```
查询选修了"数据库"课程的学生姓名:

π 姓名(σ 课程名='数据库'(学生 ⋈ 选课 ⋈ 课程))

或:
π 姓名(学生 ⋈ (σ 课程名='数据库'(课程) ⋈ 选课))
```

## 完整性约束

### 实体完整性

主键不能为空，且必须唯一。

### 参照完整性

外键必须引用存在的主键，或为空。

```
员工表:
┌─────────┬────────┬───────────┐
│ 员工ID(PK)│  姓名   │ 部门ID(FK) │
├─────────┼────────┼───────────┤
│   001   │  张三   │    10     │ ← 有效
│   002   │  李四   │    20     │ ← 有效
│   003   │  王五   │    99     │ ← 无效！部门99不存在
└─────────┴────────┴───────────┘
```

### 域完整性

属性值必须符合定义的域（数据类型、范围、格式）。

## 视图（View）

```sql
-- 创建视图
CREATE VIEW 优秀学生 AS
SELECT 学号, 姓名, 成绩
FROM 学生
WHERE 成绩 >= 90;

-- 视图是虚拟表，不存储数据
-- 可更新视图的限制较多
```

## 面试要点

### Q1: 关系 vs 非关系数据库

| 特性 | 关系数据库 | 非关系数据库 |
|------|-----------|-------------|
| 数据模型 | 表 | 文档、键值、图等 |
| Schema | 固定 | 灵活 |
| 事务 | ACID | BASE |
| 扩展 | 垂直 | 水平 |
| 查询 | SQL | 各不同 |

### Q2: 为什么需要规范化？

消除数据冗余，避免更新异常（插入、删除、修改异常）。
