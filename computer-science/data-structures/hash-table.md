# 哈希表 (Hash Table)

哈希表（散列表）是一种基于键值对的数据结构，通过哈希函数将键映射到存储位置，实现平均O(1)时间复杂度的查找、插入和删除操作。

## 核心概念

### 哈希函数
将任意大小的键转换为固定范围的整数（哈希值）：
- **确定性**：相同键产生相同哈希值
- **均匀性**：键均匀分布到哈希表槽位
- **高效性**：计算速度快

### 冲突处理
不同键映射到相同位置时的解决方法：

1. **链地址法（Separate Chaining）**
   - 每个槽位维护一个链表
   - 冲突元素加入链表
   - 查找需遍历链表

2. **开放寻址法（Open Addressing）**
   - 线性探测：顺序查找下一个空槽
   - 二次探测：按二次方步长查找
   - 双重哈希：使用第二个哈希函数

## 性能分析

| 操作 | 平均 | 最坏 |
|------|------|------|
| 插入 | O(1) | O(n) |
| 删除 | O(1) | O(n) |
| 查找 | O(1) | O(n) |

最坏情况发生在所有键都冲突时。

## 装载因子

装载因子 α = 元素数量 / 槽位数量

- α > 0.75 时考虑扩容
- 扩容需要重新哈希所有元素
- 摊销分析保证平均O(1)

## 应用场景

- 数据库索引
- 缓存实现
- 符号表（编译器）
- 去重计数
- 关联数组
