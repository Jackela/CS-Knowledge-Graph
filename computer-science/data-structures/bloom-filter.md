# 布隆过滤器 (Bloom Filter)

布隆过滤器（Bloom Filter）是一种概率型数据结构，用于高效地判断一个元素是否可能存在于集合中。它提供快速的查询和极低的内存占用，但存在一定的误判率。

## 核心概念

### 基本原理

布隆过滤器使用多个哈希函数将元素映射到一个位数组中。当查询元素时，如果所有对应位都为 1，则元素**可能**存在；如果有任意一位为 0，则元素**一定**不存在。

```
布隆过滤器结构：

初始状态（所有位为 0）：
[0, 0, 0, 0, 0, 0, 0, 0]

添加元素 "apple"（经过 3 个哈希函数）：
hash1("apple") % 8 = 1
hash2("apple") % 8 = 3  
hash3("apple") % 8 = 5

[0, 1, 0, 1, 0, 1, 0, 0]
     ↑     ↑     ↑

添加元素 "banana"：
hash1("banana") % 8 = 2
hash2("banana") % 8 = 4
hash3("banana") % 8 = 6

[0, 1, 1, 1, 1, 1, 1, 0]
        ↑     ↑     ↑

查询 "apple":
检查位 1, 3, 5 → 都为 1 → 可能存在 ✓

查询 "grape":
hash1("grape") % 8 = 1
hash2("grape") % 8 = 2
hash3("grape") % 8 = 4
检查位 1, 2, 4 → 都为 1 → 可能存在（误判！）

查询 "orange":
hash1("orange") % 8 = 7
位 7 为 0 → 一定不存在 ✓
```

### 特性

| 特性 | 说明 |
|------|------|
| **假阳性** | 可能误判元素存在（但概率可控） |
| **零假阴性** | 绝不会漏判实际存在的元素 |
| **不可删除** | 标准布隆过滤器不支持删除（可使用计数布隆过滤器） |
| **空间效率** | 远小于存储原始数据 |

## 参数配置

### 误判率公式

```
假阳性概率 ≈ (1 - e^(-kn/m))^k

其中：
m = 位数组大小
n = 预期元素数量
k = 哈希函数数量
```

### 最优参数

```
最优 k = (m/n) × ln(2)

对于给定 n 和目标误判率 p：
m = -n × ln(p) / (ln(2))^2
k = -ln(p) / ln(2)
```

| 元素数量 n | 目标误判率 | 位数组大小 m | 哈希函数 k |
|-----------|-----------|------------|-----------|
| 100万 | 1% | 9.6M bits (1.2MB) | 7 |
| 100万 | 0.1% | 14.4M bits (1.8MB) | 10 |
| 10亿 | 1% | 9.6G bits (1.2GB) | 7 |

## 应用场景

### 1. 数据库查询优化

```
使用布隆过滤器减少磁盘 I/O：

查询请求 → 布隆过滤器检查 → 不存在？直接返回
                          ↓
                    可能存在 → 查询磁盘
```

- Google Bigtable 使用布隆过滤器减少磁盘查找
- LevelDB/RocksDB 在 SSTable 中使用布隆过滤器

### 2. CDN 缓存

```
Squid Web Proxy 使用布隆过滤器：

避免缓存重复内容，快速判断 URL 是否已缓存
```

### 3. 比特币网络

```
简化支付验证（SPV）：

比特币节点使用布隆过滤器过滤交易，
仅接收与钱包相关的交易，保护隐私同时减少带宽
```

### 4. 垃圾邮件过滤

```
邮件系统使用布隆过滤器存储已知垃圾邮件地址
快速判断发件人是否为垃圾邮件源
```

## 变种与改进

### 计数布隆过滤器 (Counting Bloom Filter)

```
将位数组改为计数器数组，支持删除操作：

位数组: [1, 0, 1, 1]
计数器: [1, 0, 2, 1]  ← 记录被映射次数

删除元素时：计数器减 1，减到 0 时对应位为 0
```

### 布谷鸟过滤器 (Cuckoo Filter)

```
相比布隆过滤器的优势：
- 支持删除操作
- 更低的假阳性率
- 可以返回被存储的元素指纹
```

### 可扩展布隆过滤器 (Scalable Bloom Filter)

```
动态扩容的布隆过滤器：

当元素数量超过预期时，创建新的布隆过滤器
查询时依次检查所有过滤器
```
