# 向量数据库 (Vector Database)

## 简介

**向量数据库 (Vector Database)** 是一种专门用于存储和查询高维向量数据的数据库系统。它通过将文本、图像、音频等非结构化数据转换为向量嵌入（Embedding），实现基于语义相似度的快速检索。

```
┌─────────────────────────────────────────────────────────────┐
│                   向量数据库核心能力                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   传统数据库查询              向量数据库查询                  │
│                                                             │
│   WHERE name = 'Apple'        语义相似度搜索                  │
│                                                             │
│   精确匹配                    "苹果" 可以找到:               │
│   • 需要完全相同的值          • Apple (水果)                  │
│   • 无法理解语义              • Apple Inc (公司)              │
│   • 无法处理模糊查询          • iPhone (产品)                 │
│                               • 苹果公司 (相关概念)           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 核心概念

### 1. 向量嵌入 (Embeddings)

**嵌入 (Embedding)** 是将非结构化数据（文本、图像、音频等）映射到高维向量空间的过程，使得语义相似的对象在向量空间中距离相近。

```
嵌入过程：

文本/图像/音频
      │
      ▼
┌──────────────┐
│  嵌入模型     │  (Embedding Model)
│  • BERT      │
│  • CLIP      │
│  • OpenAI    │
│  • 自训练模型 │
└──────┬───────┘
       │
       ▼
高维向量 [0.12, -0.03, 0.89, ..., 0.45]  (通常 128-1536维)

嵌入空间示例（2D简化）：

     水果                                 科技公司
       │                                    │
       │     苹果(水果)                      │     Apple Inc
       │        ●                           │        ●
       │       /                            │       /
  橙子●      ●香蕉                          │  ●Google
       │                                   /
       │  ●葡萄                            /●Microsoft
       │                                  /
       └─────────────────────────────────▶
                    语义距离

相似概念在空间中聚集，不相似概念距离较远
```

### 2. 相似度度量 (Similarity Metrics)

向量相似度搜索需要定义距离或相似度函数，常用的有以下几种：

```
┌─────────────────────────────────────────────────────────────┐
│                   相似度计算方法                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 余弦相似度 (Cosine Similarity)                          │
│                                                             │
│              A · B           向量点积                       │
│  sim = ─────────────  =  ─────────────────                  │
│         ||A|| ||B||      向量长度乘积                       │
│                                                             │
│  范围: [-1, 1]，值越大越相似                                 │
│  特点: 关注方向，忽略幅度                                   │
│  适用: 文本相似度、推荐系统                                 │
│                                                             │
│  2. 欧几里得距离 (Euclidean Distance)                       │
│                                                             │
│  d = √Σ(Aᵢ - Bᵢ)²                                           │
│                                                             │
│  范围: [0, +∞)，值越小越相似                                 │
│  特点: 考虑向量绝对位置                                     │
│  适用: 图像特征、聚类分析                                   │
│                                                             │
│  3. 点积相似度 (Dot Product)                                │
│                                                             │
│  sim = A · B = ΣAᵢBᵢ                                        │
│                                                             │
│  范围: (-∞, +∞)                                             │
│  特点: 考虑幅度，计算最快                                   │
│  适用: 归一化后的向量                                       │
│                                                             │
│  4. 曼哈顿距离 (Manhattan Distance)                         │
│                                                             │
│  d = Σ|Aᵢ - Bᵢ|                                             │
│                                                             │
│  特点: 适合高维稀疏向量                                     │
│  适用: 文档词袋模型                                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

```python
import numpy as np

def cosine_similarity(a, b):
    """余弦相似度"""
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

def euclidean_distance(a, b):
    """欧几里得距离"""
    return np.linalg.norm(a - b)

def dot_product(a, b):
    """点积"""
    return np.dot(a, b)

# 示例
a = np.array([1, 2, 3])
b = np.array([1, 2, 3.1])

print(f"余弦相似度: {cosine_similarity(a, b):.4f}")
print(f"欧几里得距离: {euclidean_distance(a, b):.4f}")
```

### 3. 近似最近邻搜索 (ANN - Approximate Nearest Neighbor)

精确最近邻搜索在高维空间中面临"维度灾难"，时间复杂度为O(N)。ANN算法通过牺牲一定精度换取查询速度提升。

```
精确搜索 vs 近似搜索：

精确K-NN:                    近似K-NN (ANN):
                              
查询向量 q                    查询向量 q
    │                             │
    ▼                             ▼
计算到所有点的距离              使用索引结构快速定位
[计算N次距离]                   [只访问部分候选]
    │                             │
    ▼                             ▼
排序返回Top K                 返回近似Top K
                              (可能有少量误差)

时间复杂度:                    时间复杂度:
O(N × d)                      O(log N) 或 O(1)
(N=数据量, d=维度)             

适用场景:                      适用场景:
N < 10000                     N > 100000
精度要求100%                  容忍1-5%误差
```

## 索引结构

向量数据库的核心是高效的ANN索引结构。以下是主流索引算法：

### 1. HNSW (Hierarchical Navigable Small World)

**HNSW** 是一种基于图的索引结构，通过构建多层图结构实现高效导航。

```
HNSW结构示意：

Layer 2 (顶层，稀疏):         Layer 1 (中层):              Layer 0 (底层，完整):
                              
       ● A                        ● ─── ●                      ● ─── ●
      /                            \   /                         │ ╲ │
     /                              ● B                            ●
    ● B                                                        /  │  \
   /                                                          ● ─ ● ─ ●
  ● C

查询过程：
1. 从顶层随机节点开始
2. 在当前层贪心搜索最近的邻居
3. 找到最近点后，进入下一层的对应点
4. 重复直到最底层
5. 在最底层精细搜索返回Top K

特点：
• 查询速度快：O(log N)
• 内存占用较大（需要存储多层图）
• 支持增量插入
• 召回率高（>95%）
```

### 2. IVF (Inverted File Index)

**IVF** 使用聚类将向量空间划分为多个区域，查询时只搜索最相关的区域。

```
IVF结构示意：

训练阶段:                    查询阶段:
                              
1. 对数据集进行K-Means聚类    1. 计算查询向量到各中心的距离
                              
   中心点                      2. 选择最近的nprobe个桶
      ●                        
     /│\                       3. 只在选中的桶内搜索
    / │ \                         ┌─────┐
   ●  ●  ●   ← 簇中心             │ ●q  │ 查询向量
  桶1 桶2 桶3                      └──┬──┘
                                    │
   数据点                          ▼
   ○ ○ ○                           最近中心
   ○ ○ ○                              ●
   ○ ○ ○                            /   \
                                  桶A    桶B
                                  搜索   不搜索

参数：
• nlist: 桶的数量（通常√N）
• nprobe: 查询时搜索的桶数（影响精度和速度）

特点：
• 内存效率高
• 适合批量查询
• 需要预先训练
• 不支持实时插入（需要重新训练）
```

### 3. PQ (Product Quantization)

**PQ** 是一种向量压缩技术，通过将高维向量分段量化，大幅减少存储空间。

```
Product Quantization原理：

原始向量 (128维):            分段量化:
                              
[d1, d2, ..., d128]           分成m=8段，每段16维
                              
      │                       ┌────────┬────────┬────────┐
      ▼                       │段1(16d)│段2(16d)│...│段8   │
  量化编码                      └────┬───┴────┬───┴───┴──┘
                                    │        │
      ●─────────                    ▼        ▼
     /  编码本1   每个段用           ●        ●
    /   (256个    k-means训练       在各自    在各自
   ●    质心)    得到质心          编码本    编码本
      4 bytes    用质心ID代替      中找      中找
                 原始值            最近质心   最近质心

存储对比：
原始向量: 128维 × 4字节 = 512字节
PQ编码:   8段 × 1字节 = 8字节
压缩比:   64:1

特点：
• 存储空间极小
• 适合超大规模数据集（十亿级）
• 精度损失较大
• 通常与IVF结合使用（IVF-PQ）
```

### 4. pgvector

**pgvector** 是PostgreSQL的向量扩展，将向量能力带入关系型数据库。

```
pgvector使用：

-- 安装扩展
CREATE EXTENSION vector;

-- 创建向量列
CREATE TABLE items (
    id SERIAL PRIMARY KEY,
    content TEXT,
    embedding VECTOR(1536)  -- 指定维度
);

-- 创建HNSW索引
CREATE INDEX idx_embedding ON items 
USING hnsw (embedding vector_cosine_ops);

-- 向量搜索
SELECT *, embedding <=> query_embedding AS distance
FROM items
ORDER BY embedding <=> query_embedding
LIMIT 10;

特点：
• 与PostgreSQL无缝集成
• 支持ACID事务
• 支持混合查询（SQL + 向量）
• 适合已有PG基础设施的团队
```

### 4. 语义搜索

## 面试要点

### 常见问题

**Q1: 向量数据库和传统数据库的核心区别？**
> 传统数据库基于精确匹配（B+树、哈希表），适合结构化数据查询；向量数据库基于相似度搜索（ANN索引），适合非结构化数据的语义检索。传统数据库查询返回确定结果，向量数据库返回最相似的Top K。

**Q2: HNSW和IVF索引的优缺点？**
> HNSW基于图结构，支持增量插入，查询速度快，召回率高，但内存占用大；IVF基于聚类，内存效率高，但不支持实时插入，需要预先训练。HNSW适合动态数据集，IVF适合静态大批量数据。

**Q3: 如何选择向量数据库产品？**
> 考虑因素：1) 数据规模（百万级vs十亿级）2) 是否已有PG基础设施 3) 团队运维能力 4) 预算 5) 延迟要求。小规模用pgvector或Chroma；大规模生产用Milvus；快速启动用Pinecone。

**Q4: 为什么高维空间不能使用传统树结构索引？**
> 高维空间中存在"维度灾难"，数据点分布变得稀疏，距离区分度下降。传统树结构（如KD-Tree）在高维下退化为线性扫描，效率极低。ANN算法通过近似和降维来解决这个问题。

**Q5: 向量数据库中的混合搜索是什么？**
> 混合搜索结合了向量相似度搜索和元数据过滤。例如搜索"价格<100且语义相似于'舒适跑鞋'的商品"。实现方式有：1) 先向量搜索再过滤 2) 先元数据过滤再向量搜索 3) 使用支持混合索引的数据库（如Milvus、Weaviate）。

**Q6: 如何评估向量数据库性能？**
> 核心指标：1) 查询延迟（QPS/P99延迟）2) 召回率（Recall@K）3) 索引构建时间 4) 内存/磁盘占用。测试时使用标准数据集（如SIFT、GIST）和真实业务数据。
