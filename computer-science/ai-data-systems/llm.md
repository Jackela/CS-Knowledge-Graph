# 大语言模型 (Large Language Models, LLMs)

## 简介

**大语言模型 (Large Language Models, LLMs)** 是基于深度学习技术构建的人工智能系统，通过在海量文本数据上进行预训练，学习语言的模式、语法、语义和世界知识，从而能够理解和生成人类语言。

```
┌─────────────────────────────────────────────────────────────┐
│                   大语言模型核心特征                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   传统NLP:                      大语言模型:                   │
│                                                             │
│   分词 → 词性标注 → 句法分析      端到端学习                   │
│     ↓       ↓        ↓            预训练 + 微调               │
│   特征工程 → 机器学习模型         涌现能力 (Emergence)         │
│                                                             │
│   任务特定模型                    通用基础模型                 │
│   [翻译] [摘要] [分类]           [一个模型处理多种任务]        │
│                                                             │
│   参数规模: 百万级                参数规模: 十亿至万亿级         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

大语言模型的"大"体现在三个方面：
- **参数规模大**：数十亿至数万亿参数
- **训练数据大**：数万亿token的文本数据
- **计算资源大**：数千GPU/TPU训练数周至数月

## 核心概念

### 1. Transformer架构 (Transformer Architecture)

大语言模型的基础架构，详见 [Transformer](./transformers.md)。

```
Transformer架构演进:

原始Transformer (2017)
    │
    ├──▶ Encoder-only (BERT系列)
    │       └── 理解任务：分类、NER、问答
    │
    ├──▶ Decoder-only (GPT系列)
    │       └── 生成任务：文本生成、对话
    │
    └──▶ Encoder-Decoder (T5、BART)
            └── 翻译、摘要

现代LLM主要采用Decoder-only架构
```

### 2. 预训练 vs 微调 (Pre-training vs Fine-tuning)

```
┌─────────────────────────────────────────────────────────────┐
│              预训练 + 微调范式                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  阶段1: 预训练 (Pre-training)                                │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  数据: 互联网文本、书籍、代码 (数万亿token)            │   │
│  │  任务: 自监督学习 (下一个token预测)                   │   │
│  │  成本: 数百万美元计算资源                             │   │
│  │  结果: 基础模型 (Base Model)                         │   │
│  └─────────────────────────────────────────────────────┘   │
│                         │                                   │
│                         ▼                                   │
│  阶段2: 微调 (Fine-tuning)                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  数据: 特定领域标注数据 (数千至数百万样本)            │   │
│  │  任务: 监督学习 / 强化学习                            │   │
│  │  成本: 相对较小                                       │   │
│  │  结果: 专用模型 (如ChatGPT、CodeLlama)               │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**预训练任务类型：**

| 任务 | 描述 | 示例 |
|------|------|------|
| Causal LM | 自回归预测下一个token | GPT系列 |
| Masked LM | 预测被遮盖的token | BERT |
| Prefix LM | 预测给定前缀后的内容 | T5 |

### 3. Tokenization (分词)

```
文本 → Token序列 → 向量表示

原始文本: "Hello world"
    ↓
分词器 (Tokenizer)
    ↓
Tokens: ["Hello", " world"] 或 [15496, 995]
    ↓
嵌入层 (Embedding)
    ↓
向量: [[0.23, -0.45, ...], [0.12, 0.89, ...]]
```

**主流分词算法：**

```
┌─────────────────────────────────────────────────────────────┐
│                   分词算法对比                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  BPE (Byte-Pair Encoding)                                   │
│  ├── GPT系列使用                                            │
│  ├── 从字符开始，迭代合并高频对                              │
│  └── "playing" → ["play", "ing"]                            │
│                                                             │
│  WordPiece                                                    │
│  ├── BERT使用                                               │
│  ├── 基于概率选择合并                                        │
│  └── "unhappiness" → ["un", "happiness"]                    │
│                                                             │
│  SentencePiece                                                │
│  ├── T5、LLaMA使用                                          │
│  ├── 将空格视为特殊字符                                      │
│  └── 无需预分词，多语言友好                                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4. 提示工程 (Prompt Engineering)

```
提示工程：通过设计输入提示来引导模型行为

基本提示:
"翻译这句话到中文: Hello world"
    ↓
Zero-shot (零样本): 直接描述任务
"将以下英文翻译成中文: Hello world"

Few-shot (少样本): 提供示例
""
英文: Hello
中文: 你好

英文: World
中文: 世界

英文: Hello world
中文:
""

Chain-of-Thought (思维链): 引导推理
"计算 17 × 28。让我们一步一步思考。"
```

**高级提示技术：**

| 技术 | 描述 | 应用场景 |
|------|------|----------|
| Zero-shot | 直接描述任务 | 简单任务 |
| Few-shot | 提供少量示例 | 格式控制、风格迁移 |
| CoT | 引导逐步推理 | 数学、逻辑问题 |
| ReAct | 推理+行动交替 | 工具使用、Agent |
| Self-consistency | 多次采样投票 | 提高准确性 |

### 5. 上下文学习 (In-Context Learning, ICL)

```
上下文学习：模型从提示提供的示例中学习，无需更新参数

┌─────────────────────────────────────────────────────────────┐
│                   ICL 原理示意                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  输入:                                                      │
│  "判断情感倾向:                                             │
│   例子1: 这部电影太棒了! → 正面                             │
│   例子2: 完全浪费时间 → 负面                                │
│   待判断: 特效很震撼 → ?"                                   │
│                                                             │
│  模型行为:                                                  │
│  1. 通过注意力机制定位相似示例                              │
│  2. 提取任务模式 (情感分析)                                 │
│  3. 应用到新输入                                            │
│                                                             │
│  输出: "正面"                                               │
│                                                             │
│  注意: 模型权重未改变!                                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 局限与挑战

### 1. 幻觉 (Hallucination)

```
幻觉: 模型生成看似合理但实际错误的信息

类型:
├── 事实性幻觉: 编造不存在的事实
│   例: "爱因斯坦获得了诺贝尔文学奖"
│
└── 忠实性幻觉: 与输入或上下文矛盾
    例: 用户说"我不喜欢苹果", 模型回答"既然你喜欢苹果..."

原因:
├── 训练数据包含错误信息
├── 模型优化目标是流畅性而非准确性
├── 知识有截止日期
└── 对不确定性的校准不足

缓解方法:
├── RAG: 检索增强生成
├── 事实核查: 多模型验证
├── 思维链: 让模型展示推理过程
└── 人类反馈: RLHF训练
```

### 2. 偏见 (Bias)

```
偏见来源与类型:

数据偏见:
├── 训练数据反映社会既有偏见
├── 某些群体/语言数据不足
└── 历史数据包含歧视性内容

表现:
├── 性别偏见: 医生→默认男性
├── 种族偏见: 某些描述的不平等关联
├── 文化偏见: 西方中心主义视角
└── 语言偏见: 英语能力优于小语种

缓解策略:
├── 数据清洗与平衡
├── 微调去偏见
├── 输出审核机制
└── 多样化训练数据
```

### 3. 计算成本 (Computational Cost)

```
成本构成:

训练成本:
GPT-3训练: ~1200万美元 (Azure云计算)
LLaMA-2-70B: ~200万美元
更大模型: 数千万至数亿美元

推理成本:
┌──────────────┬─────────────────────────────────────┐
│ 模型规模     │ 单次推理成本估算                     │
├──────────────┼─────────────────────────────────────┤
│ 7B参数       │ ~0.0001美元/1K tokens               │
├──────────────┼─────────────────────────────────────┤
│ 70B参数      │ ~0.001美元/1K tokens                │
├──────────────┼─────────────────────────────────────┤
│ GPT-4级别    │ ~0.03美元/1K tokens                 │
└──────────────┴─────────────────────────────────────┘

优化方向:
├── 模型量化 (INT8/INT4)
├── 模型蒸馏
├── 投机解码
└── 硬件优化 (专用AI芯片)
```

### 4. 上下文长度限制 (Context Length Limitation)

```
上下文长度演进:

早期模型: 512-2048 tokens
BERT: 512
GPT-3: 2048

当前主流: 4K-128K tokens
GPT-4: 8K / 32K / 128K
Claude 3: 200K
Gemini 1.5: 1M (实验)

长上下文挑战:
├── 注意力计算复杂度 O(n²)
├── 显存占用随长度线性增长
├── 长距离注意力衰减
└── "Lost in the Middle"问题

解决方案:
├── 位置编码改进 (ALiBi, RoPE)
├── 稀疏注意力
├── 分层注意力
└── RAG外挂知识库
```

## 面试要点

### 常见问题

**Q1: LLM和传统NLP方法的主要区别是什么?**
> 传统NLP采用任务特定的流水线，需要大量特征工程和标注数据。LLM采用预训练+微调范式，通过大规模无监督学习获得通用语言能力，可通过提示快速适应新任务，具有更好的泛化能力和少样本学习能力。

**Q2: 解释RLHF (基于人类反馈的强化学习) 的原理**
> RLHF三阶段: 1) 预训练基础模型 2) 收集人类偏好数据训练奖励模型 3) 使用PPO等强化学习算法优化策略模型。奖励模型学习人类偏好，策略模型学习生成高奖励输出，使模型输出更符合人类价值观。

**Q3: 什么是涌现能力 (Emergent Abilities)?**
> 涌现能力指模型在参数规模达到某一阈值后突然表现出的新能力，如上下文学习、思维链推理。这些能力在小模型中不存在，无法通过简单外推预测，是大模型特有的现象。

**Q4: 如何缓解LLM的幻觉问题?**
> 1) RAG: 检索外部知识作为生成依据 2) 多模型验证: 交叉检查输出 3) 思维链: 要求模型展示推理过程 4) 不确定性量化: 让模型表达置信度 5) 事实核查工具: 自动验证关键事实 6) 人类反馈循环: 持续纠正错误。

**Q5: Decoder-only和Encoder-Decoder架构各有什么优势?**
> Decoder-only架构适合自回归生成任务，参数效率高，是GPT系列的选择。Encoder-Decoder架构适合序列到序列任务如翻译，编码器可以双向理解输入。现代LLM多选择Decoder-only因其简单性和可扩展性。
