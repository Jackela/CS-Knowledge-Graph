# CPU调度 (CPU Scheduling)

## 简介

**CPU调度**是操作系统的核心功能，负责从就绪队列中选择进程/线程分配CPU执行。良好的调度算法能优化系统吞吐量、响应时间和资源利用率。

## 调度指标

| 指标 | 定义 | 目标 |
|------|------|------|
| CPU利用率 | CPU忙碌时间占比 | 最大化 |
| 吞吐量 | 单位时间完成进程数 | 最大化 |
| 周转时间 | 提交到完成的时间 | 最小化 |
| 等待时间 | 在就绪队列等待时间 | 最小化 |
| 响应时间 | 提交到首次响应时间 | 最小化 |

## 调度算法

### 1. 先来先服务 (FCFS)

```
进程: P1(24ms), P2(3ms), P3(3ms)

执行顺序: P1 → P2 → P3
时间轴:   0    24   27   30

平均等待时间: (0 + 24 + 27) / 3 = 17ms
```

**缺点**：护航效应（Convoy Effect），短进程等待长进程。

### 2. 短作业优先 (SJF)

```
进程: P1(6ms), P2(8ms), P3(7ms), P4(3ms)

执行顺序: P4(3) → P1(6) → P3(7) → P2(8)
平均等待时间: (0+3+9+16)/4 = 7ms
```

**最优性**：最小化平均等待时间。

**问题**：需要预知运行时间，长进程可能饥饿。

### 3. 优先级调度

```
进程: P1(10, 优先3), P2(1, 优先1), P3(2, 优先4), P4(1, 优先5)

执行顺序（数字越小优先级越高）: P2 → P1 → P3 → P4
```

**问题**：低优先级进程饥饿。**解决**：老化（Aging）。

### 4. 时间片轮转 (Round Robin)

```
时间片 = 4ms
进程: P1(24), P2(3), P3(3)

时间轴: 0   4   7   10  14  18  22  26  30
        P1  P2  P3  P1  P1  P1  P1  P1  P1
            (完成)(完成)

上下文切换频繁，适合交互式系统
```

**时间片大小**：
- 太大 → 退化为FCFS
- 太小 → 上下文切换开销大

### 5. 多级反馈队列 (MLFQ)

```
┌─────────────┐  最高优先级，最短时间片
│  队列0      │  (如 8ms)
├─────────────┤
│  队列1      │  (如 16ms)
├─────────────┤
│  队列2      │  最低优先级，最长时间片
└─────────────┘  (如 32ms)

规则:
1. 新进程进入最高优先级队列
2. 用完时间片降级
3. 主动放弃CPU保持优先级
4. 低优先级饥饿时提升到高优先级
```

## Linux CFS调度器

**完全公平调度器（Completely Fair Scheduler）**：

- 使用红黑树管理进程
- 按虚拟运行时间（vruntime）排序
- 保证每个进程获得公平的CPU时间份额

```
vruntime = 实际运行时间 × 1024 / 进程权重

权重与nice值相关: weight = 1024 / (1.25 ^ nice)
```

## 调度算法对比

| 算法 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| FCFS | 简单 | 护航效应 | 批处理 |
| SJF | 最优等待 | 难预知 | 批处理 |
| 优先级 | 灵活 | 饥饿 | 实时系统 |
| RR | 响应好 | 切换开销 | 交互式 |
| MLFQ | 综合好 | 复杂 | 通用 |

## 面试要点

### Q1: 上下文切换的开销

保存/恢复寄存器、程序计数器、内存映射等状态。通常几微秒到几十微秒。

### Q2: 优先级反转

高优先级进程等待低优先级进程持有的资源。

**解决**：优先级继承、优先级天花板。
