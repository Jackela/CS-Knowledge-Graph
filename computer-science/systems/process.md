# 进程 (Process)

## 简介

**进程（Process）**是操作系统进行资源分配和调度的基本单位，是程序的一次执行过程。每个进程拥有独立的地址空间、内存、文件句柄等资源。

```
进程组成:
┌─────────────────────────────────────┐
│            进程控制块 (PCB)           │
│  ├─ 进程ID (PID)                     │
│  ├─ 进程状态                         │
│  ├─ 程序计数器 (PC)                  │
│  ├─ 寄存器                           │
│  ├─ 内存信息                         │
│  └─ 资源清单                         │
├─────────────────────────────────────┤
│            程序代码                   │
├─────────────────────────────────────┤
│            数据段                     │
├─────────────────────────────────────┤
│            堆栈                       │
└─────────────────────────────────────┘
```

## 进程状态

```
        创建 (New)
           │
           ▼
    ┌─────────────┐
    │   就绪      │◄────────────────┐
    │  (Ready)    │                 │
    └──────┬──────┘                 │
           │ 调度                   │
           ▼                        │
    ┌─────────────┐   I/O完成       │
    │   运行      │─────────────────┘
    │ (Running)   │
    └──────┬──────┘
           │ I/O请求
           ▼
    ┌─────────────┐
    │   等待      │
    │ (Waiting)   │
    └─────────────┘
           │
           ▼
        终止 (Terminated)
```

## 进程控制块 (PCB)

PCB是进程存在的唯一标志，包含：

- **进程标识符**：PID、父进程ID
- **处理机状态**：通用寄存器、程序计数器、PSW
- **进程控制信息**：状态、优先级、资源清单
- **内存信息**：代码段、数据段、堆栈地址

## 进程操作

### 创建

```c
// Linux fork() 示例
#include <unistd.h>
#include <sys/types.h>

pid_t pid = fork();

if (pid < 0) {
    // 创建失败
} else if (pid == 0) {
    // 子进程
    printf("Child PID: %d\n", getpid());
} else {
    // 父进程
    printf("Parent PID: %d, Child PID: %d\n", getpid(), pid);
}
```

### 终止

```c
// 正常终止
exit(0);

// 异常终止
abort();
```

### 等待

```c
// 父进程等待子进程结束
int status;
pid_t child_pid = wait(&status);
```

## 进程间通信 (IPC)

| 机制 | 特点 | 适用场景 |
|------|------|---------|
| 管道 | 半双工，亲缘进程 | 父子进程 |
| 消息队列 | 格式化消息 | 异步通信 |
| 共享内存 | 最快，需同步 | 大量数据 |
| 信号量 | 同步机制 | 临界区保护 |
| 套接字 | 跨网络 | 分布式 |

### 共享内存示例

```c
// Linux 共享内存
#include <sys/shm.h>

int shmid = shmget(IPC_PRIVATE, 1024, IPC_CREAT | 0666);
void *addr = shmat(shmid, NULL, 0);

// 读写共享内存
sprintf(addr, "Hello from process %d", getpid());

shmdt(addr);
```

## 特殊进程

### 僵尸进程 (Zombie)

子进程已终止但父进程未调用 `wait()`，PCB仍保留。

```
解决方法:
1. 父进程调用 wait()
2. 父进程终止，子进程被init收养
```

### 孤儿进程 (Orphan)

父进程先终止，子进程被init进程（PID=1）收养。

## 面试要点

### Q1: fork() 返回值

- 父进程：返回子进程PID
- 子进程：返回0
- 失败：返回-1

### Q2: 进程 vs 线程

| 特性 | 进程 | 线程 |
|------|------|------|
| 资源 | 独立 | 共享 |
| 通信 | 复杂 | 简单 |
| 切换开销 | 大 | 小 |
| 创建开销 | 大 | 小 |
| 安全性 | 高 | 低 |
