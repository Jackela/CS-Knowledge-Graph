# 回溯算法 (Backtracking)

回溯算法是一种通过穷举所有可能的候选解来找出满足条件的解的算法。当发现当前路径不可能得到解时，就回溯到上一步，尝试其他路径。

## 核心思想

### 深度优先搜索
- 沿着一条路径深入探索
- 走不通时回退，尝试其他分支

### 剪枝 (Pruning)
- 提前判断当前路径不可能产生解
- 跳过无效搜索，提高效率

### 状态恢复
- 递归返回时撤销当前选择
- 保证状态干净，不影响其他分支

## 算法框架

```
function backtrack(路径, 选择列表):
    if 满足结束条件:
        记录结果
        return
    
    for 选择 in 选择列表:
        做选择
        backtrack(路径, 新的选择列表)
        撤销选择  // 回溯
```

## 经典问题

### 排列组合
| 问题 | 关键点 |
|------|--------|
| 全排列 | 使用used数组标记已选元素 |
| 组合总和 | 注意去重，控制起始位置 |
| 子集 | 每个元素选或不选 |

### N皇后问题
- 每行每列每条对角线只能放一个皇后
- 使用三个集合记录已占用的列和对角线
- 对角线编码：row+col 和 row-col

### 数独求解
- 每个格子尝试1-9
- 检查行、列、3x3宫格是否冲突
- 找到解立即返回

### 图着色
- 相邻节点颜色不同
- 尝试m种颜色
- 判断可行性

## 复杂度分析

| 问题 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 全排列 | O(n!) | O(n) |
| 子集 | O(2^n) | O(n) |
| N皇后 | O(n!) | O(n) |

剪枝可以显著降低实际运行时间。

## 优化技巧

1. **排序预处理**：相同元素放在一起，方便去重
2. **记忆化**：记录已计算的子问题
3. **启发式排序**：优先尝试更可能成功的分支
4. **迭代加深**：限制搜索深度，逐步放宽

## 与动态规划的区别

| 特性 | 回溯 | 动态规划 |
|------|------|---------|
| 适用问题 | 求所有解/任意解 | 求最优解/解的数量 |
| 子问题 | 独立，不重叠 | 重叠子问题 |
| 时间 | 指数级 | 多项式级 |
| 空间 | 递归栈深度 | 状态表格 |
