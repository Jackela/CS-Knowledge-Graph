# 语法分析 (Parsing)

## 简介

**语法分析**是编译器的第二个阶段，负责将词法分析器产生的记号流转换为抽象语法树（AST）。语法分析器（Parser）根据上下文无关文法（Context-Free Grammar）检查记号序列是否符合语言规范，并构建出反映程序结构的树形表示。

## 核心概念

| 概念 | 英文 | 说明 |
|------|------|------|
| **CFG** | 上下文无关文法 | 定义语言语法规则的正式表示 |
| **AST** | 抽象语法树 | 源代码的抽象语法结构树 |
| **Parse Tree** | 分析树 | 完整的语法推导树 |
| **LL Parser** | 自顶向下分析器 | 从左到右扫描，最左推导 |
| **LR Parser** | 自底向上分析器 | 从左到右扫描，最右推导逆序 |
| **Recursive Descent** | 递归下降 | 基于递归函数的语法分析 |
| **Precedence Climbing** | 优先级爬升 | 处理表达式优先级的方法 |

### 1. 递归下降解析器

# recursive_descent.py
from dataclasses import dataclass
from typing import List, Optional, Union
from enum import Enum, auto

class TokenType(Enum):
    NUMBER = auto()
    PLUS = auto()
    MINUS = auto()
    MULTIPLY = auto()
    DIVIDE = auto()
    LPAREN = auto()
    RPAREN = auto()
    EOF = auto()

@dataclass
class Token:
    type: TokenType
    value: str

# AST 节点
@dataclass
class Number:
    value: float

@dataclass
class BinaryOp:
    left: 'Expr'
    op: str
    right: 'Expr'

Expr = Union[Number, BinaryOp]

class RecursiveDescentParser:
    \"\"\"递归下降解析器\"\"\"\n    \n    def __init__(self, tokens: List[Token]):\n        self.tokens = tokens\n        self.pos = 0\n    \n    def current(self) -> Token:\n        if self.pos < len(self.tokens):\n            return self.tokens[self.pos]\n        return Token(TokenType.EOF, '')\n    \n    def eat(self, token_type: TokenType) -> Token:\n        \"\"\"消费一个记号\"\"\"\n        if self.current().type == token_type:\n            token = self.current()\n            self.pos += 1\n            return token\n        raise SyntaxError(f'Expected {token_type}, got {self.current().type}')\n    \n    def parse(self) -> Expr:\n        \"\"\"入口：解析表达式\"\"\"\n        return self.expr()\n    \n    def expr(self) -> Expr:\n        \"\"\"表达式: term ((+|-) term)*\"\"\"\n        node = self.term()\n        \n        while self.current().type in (TokenType.PLUS, TokenType.MINUS):\n            op = self.current().value\n            self.eat(self.current().type)\n            right = self.term()\n            node = BinaryOp(node, op, right)\n        \n        return node\n    \n    def term(self) -> Expr:\n        \"\"\"项: factor ((*|/) factor)*\"\"\"\n        node = self.factor()\n        \n        while self.current().type in (TokenType.MULTIPLY, TokenType.DIVIDE):\n            op = self.current().value\n            self.eat(self.current().type)\n            right = self.factor()\n            node = BinaryOp(node, op, right)\n        \n        return node\n    \n    def factor(self) -> Expr:\n        \"\"\"因子: NUMBER | '(' expr ')'\"\"\"\n        token = self.current()\n        \n        if token.type == TokenType.NUMBER:\n            self.eat(TokenType.NUMBER)\n            return Number(float(token.value))\n        \n        elif token.type == TokenType.LPAREN:\n            self.eat(TokenType.LPAREN)\n            node = self.expr()\n            self.eat(TokenType.RPAREN)\n            return node\n        \n        raise SyntaxError(f'Unexpected token: {token}')\n\n# 示例\nif __name__ == '__main__':\n    # 1 + 2 * 3\n    tokens = [\n        Token(TokenType.NUMBER, '1'),\n        Token(TokenType.PLUS, '+'),\n        Token(TokenType.NUMBER, '2'),\n        Token(TokenType.MULTIPLY, '*'),\n        Token(TokenType.NUMBER, '3'),\n        Token(TokenType.EOF, '')\n    ]\n    \n    parser = RecursiveDescentParser(tokens)\n    ast = parser.parse()\n    print(ast)\n```\n\n### 2. 运算符优先级解析

       if token.type not in self.PRECEDENCE:\n                break\n            \n            precedence = self.PRECEDENCE[token.type]\n            \n            # 优先级不够高，返回\n            if next_min_precedence = precedence + 1 if token.type == TokenType.POWER else precedence\n            \n            # 解析右操作数\n            right = self.parse_expression(next_min_precedence)\n self.current()\n        \n        if token.type == TokenType.NUMBER:\n            self.eat()\n            return Number(float(token.value))\n        \n        elif token.type == TokenType.LPAREN:\n TokenType.MINUS:\n            # 一元负号\n            self.eat()\n            operand = self.parse_primary()\n            return BinaryOp(Number(0), '-', operand)\n        \n        raise Token(TokenType.EOF, '')\n    ]\n    \n    parser = PrecedenceClimbingParser(tokens)\n    ast = parser.parse()\n    print(ast)\n```\n\n### 3. 简单的 LL(1) 解析器\n\n```python\n# ll1_parser.py\nfrom LL 解析器和 LR 解析器的区别？**\nA: LL 是自顶向下（从起始符号推导出输入），从左到右扫描，最左推导。LR 是自底向上（从输入归约到起始符号），从左到右扫描，最右推导的逆序。LL 更简单直观但表达能力较弱，LR 更强大但实现复杂。LL(k) 需要 k 个 lookahead，LR(1) 通常足够。\n\n**Q: 什么是 FIRST 和 FOLLOW 集合？**\nA: FIRST(α) 是从 α 可以推导出的所有串的第一个终结符集合。FOLLOW(A) 是在句型中可以紧跟在非终结符 A 后面的所有终结符集合。它们用于构建预测分析表，决定用哪个产生式进行推导。\n\n**Q: 左递归问题如何解决？**\nA: 左递归会导致递归下降解析器无限递归。解决方法：1) 改写文法，将左递归转换为右递归；2) 使用迭代代替递归（如 E → T {+ T}）；3) 使用 LR 解析器，它能处理左递归。\n\n**Q: 如何处理二义性文法？**\nA: 二义性文法会产生多个解析树。解决方法：1) 改写文法消除二义性；2) 使用优先级和结合性规则；3) 选择特定的推导策略（如优先使用某些产生式）。\n\n**Q: 语法错误恢复策略？**\nA: 1) **恐慌模式**：跳过错误记号直到同步记号；2) **短语级恢复**：局部修改输入；3) **错误产生式**：添加处理常见错误的产生式；4) **全局纠正**：找到最小修改使输入合法。\n\n## 相关概念\n\n### 数据结构\n- [抽象语法树](../data-structures/tree.md) - AST 表示\n- [栈](../data-structures/stack.md) - 自底向上解析\n- [队列](../data-structures/queue.md)
