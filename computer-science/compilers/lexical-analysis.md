# 词法分析 (Lexical Analysis)

## 简介

**词法分析**是编译器的第一个阶段，负责将源代码字符流转换为记号（Token）序列。词法分析器（Lexer/Tokenizer）扫描源代码，识别出标识符、关键字、运算符、常量等词法单元，并去除空白字符和注释。词法分析为后续的语法分析提供标准化的输入。

## 核心概念

| 概念 | 英文 | 说明 |
|------|------|------|
| **Token** | 记号 | 词法分析的基本单位，如关键字、标识符、运算符 |
| **Lexeme** | 词素 | 记号的实际字符序列 |
| **Pattern** | 模式 | 描述词素结构的规则 |
| **Regular Expression** | 正则表达式 | 定义词法模式的工具 |
| **Finite Automaton** | 有限自动机 | 识别正则语言的计算模型 |
| **DFA** | 确定有限自动机 | 每个状态对每个输入有唯一转移 |
| **NFA** | 非确定有限自动机 | 允许ε转移和多重转移 |

    ('END',      r';'),                     # 语句结束

    ('MISMATCH', r'.'),                     # 任何其他字符
]

token_regex = '|'.join(f'(?P<{name}>{pattern})' 
                       for name, pattern in TOKEN_SPECIFICATION)

词法分析将字符流转换为记号流，关注的是词法单元的识别（正则语言层面）。语法分析将记号流转换为语法树，关注的是语言结构（上下文无关文法层面）。词法分析是编译器的第一个阶段，输出作为语法分析的输入。\n\n**Q: 为什么要将词法分析和语法分析分成两个阶段？**\nA: 1) **简化设计**：每个阶段职责单一，易于实现和维护；2) **提高效率**：词法分析可以使用高效的 DFA，语法分析使用更强大的下推自动机；3) **更好的错误处理**：可以分别定位词法错误和语法错误；4) **模块化**：词法规则变化不影响语法分析器。\n\n**Q: 正则表达式和有限自动机的关系？**\nA: 正则表达式和有限自动机是等价的，可以相互转换。正则表达式描述词法模式，有限自动机实现模式匹配。Thompson 构造法可以将正则表达式转换为 NFA，子集构造法可以将 NFA 转换为 DFA，DFA 最小化可以优化自动机。\n\n**Q: 如何处理词法分析中的冲突？**\nA: 常见冲突及解决方案：1) **最长匹配原则**：选择最长的匹配（如 `>=` 优先于 `>`）；2) **优先级规则**：先定义的规则优先；3) **关键字 vs 标识符**：先查关键字表，再作为标识符；4) **二义性**：通过上下文或额外规则解决。\n\n**Q: 词法分析中的错误恢复策略？**\nA: 1) **恐慌模式**：跳过错误字符直到找到合法记号；2) **错误记号**：生成特殊的错误记号；3) **行号列号**：精确定位错误位置；4) **错误报告**：提供有用的错误信息；5) **容错处理**：如自动补全引号。\n\n## 相关概念\n\n### 数据结构\n- [有限自动机](../data-structures/graph.md) -
