# 服务网格 (Service Mesh)

## 概念

服务网格（Service Mesh）是一种**基础设施层**，用于处理服务间的通信，提供负载均衡、服务发现、加密、可观测性等功能。

> **核心思想**: 将服务通信逻辑从业务代码中剥离，下沉到基础设施层。

## 架构

```
┌─────────────────────────────────────────────┐
│              Service Mesh 架构               │
├─────────────────────────────────────────────┤
│                                             │
│   ┌─────────┐         ┌─────────┐          │
│   │ Service │◄───────►│ Service │          │
│   │    A    │  Envoy  │    B    │          │
│   └────┬────┘ Sidecar └────┬────┘          │
│        │                   │               │
│        └───────┬───────────┘               │
│                │                           │
│                ▼                           │
│         Control Plane                      │
│         (Istiod/Linkerd)                   │
│                │                           │
│     ┌─────────┼─────────┐                  │
│     ▼         ▼         ▼                  │
│  Config    Discovery   Certificates        │
│                                             │
└─────────────────────────────────────────────┘
```

## 核心功能

| 功能 | 说明 |
|------|------|
| **流量管理** | 路由、分流、熔断、重试 |
| **安全** | mTLS、认证、授权 |
| **可观测性** | 指标、日志、追踪 |
| **策略执行** | 访问控制、速率限制 |

## 主流实现

| 方案 | 特点 |
|------|------|
| **Istio** | 功能最全，与K8s深度集成 |
| **Linkerd** | 轻量级，易于使用 |
| **Consul Connect** | HashiCorp生态 |
| **AWS App Mesh** | 云原生，托管服务 |

## Sidecar模式

服务网格使用Sidecar代理拦截所有网络流量：

```
Pod:
┌─────────────────────────┐
│  ┌─────┐   ┌────────┐  │
│  │ App │◄─►│ Envoy  │  │
│  │容器 │   │ Sidecar│  │
│  └──┬──┘   └───┬────┘  │
│     └──────────┘       │
└─────────────────────────┘
```
