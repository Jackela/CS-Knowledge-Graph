# CQRS 模式 (Command Query Responsibility Segregation)

## 概念

CQRS（命令查询职责分离）是一种**架构模式**，将数据读取（Query）和写入（Command）操作分离到不同的模型中。

> **核心思想**: 读写分离，针对不同操作优化各自的数据模型。

## 为什么需要CQRS

### 传统模式的痛点
```
单一模型同时承担读写职责：
- 读操作需要扁平化、聚合的数据
- 写操作需要保证一致性和完整性
- 两种需求导致模型复杂且低效
```

### CQRS解决方案
```
┌─────────────────┐     ┌─────────────────┐
│   Command Side  │     │    Query Side   │
│   (写入模型)     │     │    (读取模型)    │
├─────────────────┤     ├─────────────────┤
│ • 领域模型       │     │ • 视图模型       │
│ • 业务规则验证   │     │ • 数据扁平化     │
│ • 事件发布       │     │ • 查询优化       │
└────────┬────────┘     └────────┬────────┘
         │                       │
         ▼                       ▼
┌─────────────────┐     ┌─────────────────┐
│   写入数据库     │────▶│   读取数据库     │
│  (事务一致性)    │同步/ │  (最终一致性)    │
│                 │ 异步  │                 │
└─────────────────┘     └─────────────────┘
```

## 使用场景

| 场景 | 说明 |
|------|------|
| 读多写少 | 读取频率远高于写入 |
| 复杂查询 | 需要多表关联、聚合统计 |
| 高性能读 | 读取延迟要求高 |
| 团队协作 | 读写团队分离开发 |

## 优缺点

**优点**:
- 读写可以独立优化
- 查询性能大幅提升
- 读写可以独立扩展

**缺点**:
- 系统复杂度增加
- 数据一致性挑战
- 需要处理同步延迟
