# 时间复杂度

**时间复杂度**（Time Complexity）描述算法执行时间随输入规模增长的变化趋势。它不是实际运行时间，而是**问题规模 n 趋近于无穷大时，算法执行次数的数量级**。

计算机科学中，我们用时间复杂度来比较算法的效率，而不必关心具体硬件和实现细节。

```
为什么需要时间复杂度?

┌─────────────────────────────────────────────────┐
│  同一算法在不同机器上运行时间不同                 │
│  同一机器运行同一算法多次时间也可能不同           │
│                                                 │
│  时间复杂度 → 剥离硬件/环境差异，描述本质效率     │
└─────────────────────────────────────────────────┘
```

## 大O表示法 (Big-O Notation)

大O表示法描述算法的**渐近上界**，表示最坏情况下的复杂度。

### 常见时间复杂度

```
复杂度增长曲线对比:

时间
 │                                          ╱ O(n!)
 │                                       ╱
 │                                    ╱    ╱ O(2ⁿ)
 │                                 ╱    ╱
 │                              ╱    ╱
 │                           ╱    ╱        ╱ O(n²)
 │                        ╱    ╱       ╱
 │                     ╱    ╱      ╱
 │                  ╱    ╱     ╱
 │               ╱    ╱    ╱         ╱ O(n log n)
 │            ╱    ╱   ╱        ╱
 │         ╱    ╱  ╱       ╱
 │      ╱    ╱ ╱     ╱
 │   ╱    ╱╱    ╱
 │╱   ╱╱   ╱            ╱ O(n)
 │╱  ╱              ╱
 │╱            ╱
 │       ╱              ╱ O(log n)
 │  ╱
 │╱                     ╱ O(1)
 └───────────────────────────────────────> 输入规模 n
```

| 复杂度 | 名称 | 可处理规模 | 示例 |
|--------|------|-----------|------|
| **O(1)** | 常数时间 | 任意 | 数组随机访问、哈希表查询 |
| **O(log n)** | 对数时间 | 极大 | 二分查找、平衡树操作 |
| **O(n)** | 线性时间 | 较大 | 遍历数组、线性查找 |
| **O(n log n)** | 线性对数 | 中等 | 快速排序、归并排序 |
| **O(n²)** | 平方时间 | 较小 | 冒泡排序、双重循环 |
| **O(2ⁿ)** | 指数时间 | 很小 | 递归求解子集问题 |
| **O(n!)** | 阶乘时间 | 极小 | 全排列、旅行商问题 |

### O(1) - 常数时间

```
操作次数与输入规模无关:

输入规模 n    操作次数
   10    ────→   1
  100    ────→   1
 1000    ────→   1
10000    ────→   1

代码示例:
┌────────────────────────┐
│  int getFirst(int[] a) │
│  {                     │
│    return a[0];   ← O(1)│
│  }                     │
└────────────────────────┘
```

### O(log n) - 对数时间

```
操作次数随 n 增长很慢:

输入规模 n    操作次数
   10    ────→   4
  100    ────→   7
 1000    ────→  10
10000    ────→  14  (log₂10000 ≈ 14)

二分查找示例:
┌────────────────────────────────┐
│  [1, 3, 5, 7, 9, 11, 13, 15]   │
│   ↑                    ↑       │
│  low                 high      │
│                                 │
│  查找 11:                        │
│  第1轮: mid=7, 11>7, 取右半部分  │
│  第2轮: mid=13, 11<13, 取左半部分│
│  第3轮: mid=11, 找到!           │
│                                 │
│  n=8, 只比较了3次 = log₂8       │
└────────────────────────────────┘
```

### O(n) - 线性时间

```
操作次数与 n 成正比:

输入规模 n    操作次数
   10    ────→  10
  100    ────→ 100
 1000    ────→ 1000

代码示例:
┌─────────────────────────────────┐
│  int findMax(int[] a) {         │
│    int max = a[0];              │
│    for (int x : a) {      ← O(n)│
│      if (x > max) max = x;      │
│    }                            │
│    return max;                  │
│  }                              │
└─────────────────────────────────┘
```

### O(n log n) - 线性对数时间

```
代码示例 (归并排序):
┌─────────────────────────────────────────┐
│  mergeSort(arr, left, right) {          │
│    if (left < right) {                  │
│      mid = (left + right) / 2;          │
│      mergeSort(arr, left, mid);    ←递归 │
│      mergeSort(arr, mid+1, right);  log n层│
│      merge(arr, left, mid, right);  O(n) │
│    }                                    │
│  }                                      │
│                                         │
│  递归深度: log n 层                      │
│  每层工作量: O(n)                        │
│  总复杂度: O(n log n)                    │
└─────────────────────────────────────────┘
```

### O(n²) - 平方时间

```
操作次数随 n 平方增长:

输入规模 n    操作次数
   10    ────→  100
  100    ────→  10000
 1000    ────→  1000000

代码示例 (冒泡排序):
┌──────────────────────────────────────────┐
│  bubbleSort(int[] a) {                   │
│    for (i = 0; i < n; i++) {       ← n次  │
│      for (j = 0; j < n-i-1; j++) { ← n次  │
│        if (a[j] > a[j+1]) swap;          │
│      }                                   │
│    }                                     │
│  }                                       │
│                                          │
│  比较次数 ≈ n + (n-1) + ... + 1 = n(n+1)/2 │
│  复杂度: O(n²)                           │
└──────────────────────────────────────────┘
```

### O(2ⁿ) - 指数时间

```
操作次数爆炸性增长:

输入规模 n    操作次数
   10    ────→  1024
   20    ────→  100万
   30    ────→  10亿

代码示例 (斐波那契递归):
┌──────────────────────────────────────┐
│  fib(n) {                            │
│    if (n <= 1) return n;             │
│    return fib(n-1) + fib(n-2);  ← 2ⁿ │
│  }                                   │
│                                      │
│  递归树:                              │
│       fib(5)                         │
│      /        \                     │
│   fib(4)     fib(3)                  │
│   /    \     /    \                 │
│ fib(3) fib(2) fib(2) fib(1)          │
│                                      │
│  节点数 ≈ 2ⁿ                         │
└──────────────────────────────────────┘
```

### 图算法

```
图算法复杂度:

DFS/BFS: O(V + E)
├─ V: 顶点数
├─ E: 边数
└─ 每条边和顶点访问一次

Dijkstra (优先队列): O((V+E) log V)
├─ 每个顶点出堆一次: O(V log V)
└─ 每条边松弛一次: O(E log V)

Floyd-Warshall: O(V³)
├─ 三重循环
└─ 求所有点对最短路径

拓扑排序: O(V + E)
└─ Kahn算法或DFS
```

## 最好/最坏/平均情况

### 三种情况的定义

```
不同输入导致的性能差异:

┌──────────────────────────────────────────────────────┐
│                  算法: 快速排序                       │
├──────────────────────────────────────────────────────┤
│ 最好情况: 每次分区都均匀分成两半                       │
│          → 递归深度 log n, 复杂度 O(n log n)         │
│                                                      │
│ 最坏情况: 数组已有序，每次分区极不平衡                 │
│          → 递归深度 n, 复杂度 O(n²)                  │
│                                                      │
│ 平均情况: 随机输入                                    │
│          → 期望复杂度 O(n log n)                     │
└──────────────────────────────────────────────────────┘
```

### 分析示例

```
在数组中查找元素 x:

代码:
┌─────────────────────────────────┐
│  int search(int[] a, int x) {   │
│    for (int i = 0; i < n; i++) │
│      if (a[i] == x) return i;   │
│    return -1;                   │
│  }                              │
└─────────────────────────────────┘

最好情况: x 在数组第一个位置
          → 比较 1 次 → O(1)

最坏情况: x 不在数组中 或 在最后一个位置
          → 比较 n 次 → O(n)

平均情况: x 等概率出现在任意位置
          → 期望比较次数 = (1+2+...+n)/n = (n+1)/2 → O(n)
```

## 时间复杂度分析技巧

### 规则速查

```
分析法则:

1. 忽略常数项
   O(2n) = O(n)
   O(100) = O(1)

2. 忽略低阶项
   O(n² + n) = O(n²)
   O(n³ + n² + n) = O(n³)

3. 加法法则: 取最大值
   O(f(n)) + O(g(n)) = O(max(f(n), g(n)))

4. 乘法法则: 相乘
   O(f(n)) × O(g(n)) = O(f(n) × g(n))

常见结构复杂度:
┌─────────────────┬────────────────┐
│ 单层循环        │ O(n)           │
│ 双重循环        │ O(n²)          │
│ 循环折半        │ O(log n)       │
│ 递归折半        │ O(log n)       │
│ 递归分治(n/2)   │ O(n log n)     │
│ 递归分治(n-1)   │ O(n) 或 O(2ⁿ)  │
└─────────────────┴────────────────┘
```

### 递归复杂度: 主定理

```
主定理 (Master Theorem):

对于形如 T(n) = aT(n/b) + f(n) 的递归式:

其中:
├─ a: 子问题数量
├─ n/b: 子问题规模
└─ f(n): 分解与合并的代价

三种情况:

情况1: f(n) = O(n^c), c < log_b(a)
       → T(n) = Θ(n^(log_b a))
       
       例: T(n) = 2T(n/2) + O(1)
           a=2, b=2, log₂2 = 1
           f(n)=O(1)=O(n⁰), c=0 < 1
           → T(n) = Θ(n)

情况2: f(n) = Θ(n^c log^k n), c = log_b(a)
       → T(n) = Θ(n^c log^(k+1) n)
       
       例: T(n) = 2T(n/2) + O(n)
           a=2, b=2, log₂2 = 1
           f(n)=O(n), c=1 = 1
           → T(n) = Θ(n log n)

情况3: f(n) = Ω(n^c), c > log_b(a)
       → T(n) = Θ(f(n))
       
       例: T(n) = 2T(n/2) + O(n²)
           a=2, b=2, log₂2 = 1
           f(n)=O(n²), c=2 > 1
           → T(n) = Θ(n²)
```

## 面试要点

### Q1: O(n) 和 O(2n) 有什么区别?

**答案**: 在大O表示法中，O(n) 和 O(2n) 是等价的。大O描述的是**增长趋势**，不是具体系数。我们忽略常数因子，只关注最高阶项。

### Q2: 为什么说哈希表查询是 O(1)?

**答案**:
- 理想情况下，哈希函数将键均匀分布到桶中
- 通过哈希函数直接计算出存储位置，无需遍历
- 冲突处理（如链地址法）在负载因子合理时，查找仍为 O(1)
- 最坏情况（所有键冲突）退化为 O(n)，但实际很少发生

### Q3: 快速排序最坏情况 O(n²)，为什么还说它比 O(n log n) 的归并排序快?

**答案**:
1. 平均情况下两者都是 O(n log n)
2. 快速排序常数因子更小，实际运行更快
3. 快速排序是原地排序，空间复杂度 O(log n)
4. 归并排序需要 O(n) 额外空间
5. 随机化快速排序可避免最坏情况

### Q4: 如何优化双重循环?

**答案**:
```
常见优化策略:

1. 哈希表优化: O(n²) → O(n)
   例: 两数之和，用哈希表存储已遍历元素

2. 双指针: O(n²) → O(n)
   例: 有序数组求和，左右指针向中间移动

3. 排序+双指针: O(n²) → O(n log n)
   例: 三数之和，先排序再用双指针

4. 分治: O(n²) → O(n log n)
   例: 最近点对问题
```

### Q5: 递归算法的时间复杂度怎么分析?

**答案**:
```
方法1: 画递归树
       计算每层工作量 × 层数

方法2: 主定理 (Master Theorem)
       适用于 T(n) = aT(n/b) + f(n)

方法3: 代入法
       猜测上界，用数学归纳法证明

方法4: 迭代展开
       逐层展开递归式，找规律
```
