## 简介

**密码学 (Cryptography)** 是研究如何在敌对环境（存在窃听者和攻击者）中实现安全通信的数学科学。其核心目标是保护信息的机密性、完整性和真实性，同时提供不可否认性。

```
┌─────────────────────────────────────────────────────────────────┐
│                   密码学的核心目标 (CIA + A + N)                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────┐  ┌─────────────┐  ┌─────────────┐            │
│   │ Confidentiality │  Integrity    │  Authenticity │           │
│   │   机密性        │   完整性      │    真实性      │           │
│   └─────────────┘  └─────────────┘  └─────────────┘            │
│          │                │                │                   │
│          ▼                ▼                ▼                   │
│   ┌─────────────┐  ┌─────────────┐  ┌─────────────┐            │
│   │ 加密技术     │  │ 哈希/MAC     │  │ 数字签名     │            │
│   │ Encryption  │  │ Hash/MAC    │  │ Signature   │            │
│   └─────────────┘  └─────────────┘  └─────────────┘            │
│                                                                 │
│   + Non-repudiation (不可否认性): 防止发送方否认已发送消息      │
│   + Authentication (身份认证): 验证通信双方身份                  │
│                                                                 │
│   示例场景：                                                     │
│   • Alice 发送加密消息给 Bob                                     │
│   • Eve 可以看到密文，但无法读取内容 (机密性)                    │
│   • 消息被篡改会被发现 (完整性)                                  │
│   • Bob 确认消息确实来自 Alice (真实性)                          │
│   • Alice 无法否认发送过该消息 (不可否认性)                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 密码学分支

### 1. 对称加密 (Symmetric Encryption)

对称加密使用**相同的密钥**进行加密和解密，运算速度快，适合加密大量数据。

```
┌─────────────────────────────────────────────────────────────────┐
│                   对称加密原理                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   明文 (Plaintext)                                              │
│      │                                                          │
│      ▼                                                          │
│   ┌─────────────┐                                               │
│   │  加密算法    │◀──── 共享密钥 (Shared Key)                    │
│   └─────────────┘                                               │
│      │                                                          │
│      ▼                                                          │
│   密文 (Ciphertext) ──────▶ 传输 ──────▶ [解密] ────▶ 明文      │
│                                                                 │
│   特点：                                                         │
│   • 加密解密使用同一密钥                                         │
│   • 运算速度快（适合大数据）                                     │
│   • 密钥分发是主要挑战                                           │
│                                                                 │
│   代表算法：AES, ChaCha20, DES(已淘汰)                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

详见 [对称加密](./cryptography/symmetric-encryption.md)

### 2. 非对称加密 (Asymmetric Encryption)

非对称加密使用**密钥对**（公钥和私钥），公钥可公开，私钥保密。解决了密钥分发问题，但运算较慢。

```
┌─────────────────────────────────────────────────────────────────┐
│                   非对称加密原理                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   密钥对生成：                                                   │
│   ┌─────────────┐                                               │
│   │  密钥生成器  │───▶ 公钥 (Public Key) ───▶ 公开分发           │
│   │  Key Gen    │───▶ 私钥 (Private Key) ──▶ 保密存储           │
│   └─────────────┘                                               │
│                                                                 │
│   加密过程：                                                     │
│   明文 ───▶ [公钥加密] ───▶ 密文 ───▶ 传输 ───▶ [私钥解密]      │
│                                                          ▼      │
│                                                     明文        │
│                                                                 │
│   特点：                                                         │
│   • 使用密钥对（公钥 + 私钥）                                    │
│   • 解决了密钥分发问题                                           │
│   • 运算速度慢（比对称加密慢100-1000倍）                         │
│                                                                 │
│   代表算法：RSA, ECC, ElGamal                                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

详见 [非对称加密](./cryptography/asymmetric-encryption.md)

### 3. 哈希函数 (Hash Functions)

哈希函数将**任意长度输入**转换为**固定长度输出**，具有单向性和抗碰撞性。

```
┌─────────────────────────────────────────────────────────────────┐
│                   哈希函数原理                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   输入（任意长度）                                               │
│   ┌───────────────────────────────────────────────────────┐    │
│   │ "Hello, World!"                                       │    │
│   │ "The quick brown fox..."                              │    │
│   │ 二进制文件（GB级）                                     │    │
│   └───────────────────────────────────────────────────────┘    │
│                            │                                    │
│                            ▼                                    │
│                    ┌───────────────┐                           │
│                    │   哈希函数     │                           │
│                    │   Hash()      │                           │
│                    └───────┬───────┘                           │
│                            │                                    │
│                            ▼                                    │
│   输出（固定长度）                                              │
│   ┌───────────────────────────────────────────────────────┐    │
│   │ SHA-256: a591a6d40bf420404a011733cfb7b190d62c65bf...   │    │
│   └───────────────────────────────────────────────────────┘    │
│                                                                 │
│   安全性质：                                                     │
│   • 单向性：无法从哈希值反推输入                                │
│   • 抗碰撞：难以找到两个不同输入产生相同输出                    │
│   • 雪崩效应：输入微小改变导致输出大幅变化                      │
│                                                                 │
│   代表算法：SHA-256, SHA-3, BLAKE3                              │
│   ❌ 已淘汰：MD5, SHA-1                                         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

详见 [哈希函数](./cryptography/hash-functions.md)

### 4. 数字签名 (Digital Signatures)

数字签名结合哈希函数和非对称加密，提供**身份认证**、**完整性验证**和**不可否认性**。

```
┌─────────────────────────────────────────────────────────────────┐
│                   数字签名原理                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   签名过程：                                                     │
│   ┌───────────┐    ┌───────────┐    ┌───────────┐              │
│   │  消息 M   │───▶│  Hash(M)  │───▶│  私钥签名  │              │
│   └───────────┘    └───────────┘    └─────┬─────┘              │
│                                           │                     │
│                                           ▼                     │
│   发送: [消息 M] + [数字签名]                                   │
│                                                                 │
│   验证过程：                                                     │
│   ┌───────────┐    ┌───────────┐    ┌───────────┐              │
│   │  消息 M   │───▶│  Hash(M)  │    │  公钥验证  │              │
│   └───────────┘    └───────────┘    └─────┬─────┘              │
│                                           │                     │
│   签名 ──────────────────────────────────▶│                     │
│                                           ▼                     │
│                                        [有效/无效]              │
│                                                                 │
│   应用：代码签名、软件更新验证、区块链交易、文档签署              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

详见 [数字签名](./cryptography/digital-signatures.md)

## 关键概念

### 密钥 (Key)

密钥是密码学中的核心要素，决定了加密算法的安全性。

```
┌─────────────────────────────────────────────────────────────────┐
│                   密钥类型对比                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────┐  ┌─────────────────┐                     │
│   │   对称密钥       │  │   非对称密钥     │                     │
│   ├─────────────────┤  ├─────────────────┤                     │
│   │ 单密钥           │  │ 密钥对           │                     │
│   │ 加密解密都用它   │  │ 公钥加密/验签   │                     │
│   │                  │  │ 私钥解密/签名   │                     │
│   ├─────────────────┤  ├─────────────────┤                     │
│   │ 长度：128-256位 │  │ RSA: 2048-4096位│                     │
│   │ AES-256-GCM     │  │ ECC: 256-521位  │                     │
│   ├─────────────────┤  ├─────────────────┤                     │
│   │ 速度快           │  │ 速度慢           │                     │
│   │ 适合大数据       │  │ 适合小数据       │                     │
│   └─────────────────┘  └─────────────────┘                     │
│                                                                 │
│   密钥管理原则：                                                 │
│   • 使用密码学安全的随机数生成器 (CSPRNG)                       │
│   • 密钥长度要足够（考虑量子计算威胁）                          │
│   • 定期轮换密钥                                                │
│   • 使用密钥管理系统 (KMS) 或硬件安全模块 (HSM)                │
│   • 绝不硬编码密钥在源代码中                                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 明文与密文 (Plaintext & Ciphertext)

```
┌─────────────────────────────────────────────────────────────────┐
│                   明文与密文                                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   明文 (Plaintext): 未加密的原始数据                             │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │  "转账 1000 元给 Bob"                                    │   │
│   │  {"user": "alice", "amount": 1000, "to": "bob"}         │   │
│   └─────────────────────────────────────────────────────────┘   │
│                              │                                  │
│                              ▼                                  │
│                         [加密算法 + 密钥]                       │
│                              │                                  │
│                              ▼                                  │
│   密文 (Ciphertext): 加密后的数据                               │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │  0x7f3a9b2c4d5e6f7890abcdef1234567890abcdef...         │   │
│   │  （对人类不可读，需要密钥解密）                          │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│   注意：                                                         │
│   • 密文应保持随机性（相同明文产生不同密文）                    │
│   • 密文长度通常大于或等于明文长度                              │
│   • 密文本身不提供完整性保护（需要MAC或认证加密）               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 密码分析 (Cryptanalysis)

密码分析是研究如何破解密码系统的学科，了解攻击方法有助于设计更安全的系统。

```
┌─────────────────────────────────────────────────────────────────┐
│                   常见密码攻击类型                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   1. 暴力破解 (Brute Force Attack)                              │
│      • 尝试所有可能的密钥组合                                   │
│      • 安全性取决于密钥空间大小                                 │
│      • 防御：使用足够长的密钥，增加迭代成本                     │
│                                                                 │
│   2. 已知明文攻击 (Known Plaintext Attack)                      │
│      • 攻击者知道一些明文-密文对                                │
│      • 分析加密模式，推导密钥                                   │
│      • 防御：使用强加密模式（如GCM），随机IV                    │
│                                                                 │
│   3. 选择明文攻击 (Chosen Plaintext Attack)                     │
│      • 攻击者可以选择明文并获得对应密文                         │
│      • 用于分析加密系统的弱点                                   │
│      • 防御：语义安全（相同明文产生不同密文）                   │
│                                                                 │
│   4. 选择密文攻击 (Chosen Ciphertext Attack)                    │
│      • 攻击者可以选择密文并获得解密结果                         │
│      • 用于攻击 padding oracle 等漏洞                           │
│      • 防御：认证加密 (AEAD)，正确处理解密失败                  │
│                                                                 │
│   5. 中间人攻击 (Man-in-the-Middle Attack)                      │
│      • 攻击者截获并可能修改通信                                 │
│      • 防御：数字证书、证书固定 (Pinning)                       │
│                                                                 │
│   6. 重放攻击 (Replay Attack)                                   │
│      • 截获并重新发送之前的有效消息                             │
│      • 防御：使用随机数 (Nonce)、时间戳、序列号                 │
│                                                                 │
│   7. 侧信道攻击 (Side-Channel Attack)                           │
│      • 分析功耗、时序、电磁泄漏等信息                           │
│      • 防御：恒定时间算法、随机延迟、硬件防护                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 应用场景

### 1. HTTPS/TLS 安全通信

HTTPS 结合了对称加密、非对称加密和哈希函数，保护 Web 通信安全。

```
┌─────────────────────────────────────────────────────────────────┐
│                   HTTPS/TLS 握手过程                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   客户端                          服务器                         │
│   ─────                          ─────                          │
│      │                              │                           │
│      │  1. ClientHello              │                           │
│      │     - 支持的加密套件         │                           │
│      │     - 随机数 (Client Random) │                           │
│      │─────────────────────────────▶│                           │
│      │                              │                           │
│      │  2. ServerHello              │                           │
│      │     - 选择的加密套件         │                           │
│      │     - 随机数 (Server Random) │                           │
│      │                              │                           │
│      │  3. Certificate              │                           │
│      │     - 服务器证书（含公钥）   │                           │
│      │◀─────────────────────────────│                           │
│      │                              │                           │
│      │  4. 验证证书                 │                           │
│      │     - 检查签名               │                           │
│      │     - 验证证书链             │                           │
│      │                              │                           │
│      │  5. 密钥交换                 │                           │
│      │     - 生成预主密钥           │                           │
│      │     - 用服务器公钥加密传输   │                           │
│      │─────────────────────────────▶│                           │
│      │                              │                           │
│      │  6. 生成会话密钥             │                           │
│      │     - 双方使用相同算法       │                           │
│      │     - 派生对称加密密钥       │                           │
│      │                              │                           │
│      │  ═══════════════════════════════════════                 │
│      │  握手完成！后续使用对称加密通信                           │
│      │  ═══════════════════════════════════════                 │
│      │                              │                           │
│                                                                 │
│   使用的密码学技术：                                             │
│   • 非对称加密：密钥交换、证书验证                              │
│   • 对称加密：AES-GCM/ChaCha20-Poly1305 加密应用数据           │
│   • 哈希函数：证书签名验证、密钥派生                            │
│   • 数字签名：证书签发验证                                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

详见 [HTTPS/TLS](../computer-science/networks/https-tls.md)

### 2. 数字证书与 PKI

数字证书将公钥与身份绑定，由可信的证书颁发机构 (CA) 签发。

```
┌─────────────────────────────────────────────────────────────────┐
│                   数字证书体系 (PKI)                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────┐                                               │
│   │   根 CA      │  ─── 自签名证书                               │
│   │  Root CA    │                                               │
│   └──────┬──────┘                                               │
│          │                                                      │
│          ▼                                                      │
│   ┌─────────────┐                                               │
│   │  中间 CA     │  ─── 由根CA签名                               │
│   │ Intermediate│                                               │
│   └──────┬──────┘                                               │
│          │                                                      │
│          ▼                                                      │
│   ┌─────────────┐                                               │
│   │  服务器证书  │  ─── 由中间CA签名                             │
│   │ Server Cert │                                               │
│   └─────────────┘                                               │
│                                                                 │
│   证书内容：                                                     │
│   • 持有者信息（域名、组织等）                                  │
│   • 公钥                                                        │
│   • 有效期                                                      │
│   • 颁发者信息                                                  │
│   • 数字签名（用CA私钥签名）                                    │
│                                                                 │
│   验证过程：                                                     │
│   1. 检查证书有效期                                             │
│   2. 验证证书签名（用CA公钥）                                   │
│   3. 验证证书链直到受信任的根CA                                 │
│   4. 检查证书吊销状态 (CRL/OCSP)                                │

## 代码示例

### Python 基础加密操作

```python
import hashlib
import secrets
import base64
from cryptography.fernet import Fernet
from cryptography.hazmat.primitives import hashes
from cryptography.hazmat.primitives.kdf.pbkdf2 import PBKDF2

# ========== 1. 哈希计算 ==========
def compute_hash(data: bytes, algorithm: str = 'sha256') -> str:
    """计算数据的哈希值"""
    if algorithm == 'sha256':
        return hashlib.sha256(data).hexdigest()
    elif algorithm == 'sha3_256':
        return hashlib.sha3_256(data).hexdigest()
    else:
        raise ValueError(f"不支持的算法: {algorithm}")

# 示例
data = b"Hello, Cryptography!"
print(f"SHA-256: {compute_hash(data)}")

# ========== 2. 对称加密 (Fernet) ==========
def generate_key() -> bytes:
    """生成对称加密密钥"""
    return Fernet.generate_key()

def symmetric_encrypt(plaintext: bytes, key: bytes) -> bytes:
    """对称加密"""
    f = Fernet(key)
    return f.encrypt(plaintext)

def symmetric_decrypt(ciphertext: bytes, key: bytes) -> bytes:
    """对称解密"""
    f = Fernet(key)
    return f.decrypt(ciphertext)

# 示例
key = generate_key()
plaintext = b"Secret message"
ciphertext = symmetric_encrypt(plaintext, key)
decrypted = symmetric_decrypt(ciphertext, key)
print(f"加密: {ciphertext}")
print(f"解密: {decrypted}")

# ========== 3. 安全随机数 ==========
def generate_secure_random(length: int = 32) -> bytes:
    """生成密码学安全的随机数"""
    return secrets.token_bytes(length)

def generate_secure_token() -> str:
    """生成安全的URL安全令牌"""
    return secrets.token_urlsafe(32)

# 示例
random_bytes = generate_secure_random(16)
print(f"随机数: {random_bytes.hex()}")
print(f"令牌: {generate_secure_token()}")

# ========== 4. 密钥派生 (PBKDF2) ==========
def derive_key(password: str, salt: bytes, iterations: int = 100000) -> bytes:
    """从密码派生加密密钥"""
    kdf = PBKDF2(
        algorithm=hashes.SHA256(),
        length=32,
        salt=salt,
        iterations=iterations,
    )
    return kdf.derive(password.encode())

# 示例
password = "user_password"
salt = secrets.token_bytes(16)
key = derive_key(password, salt)
print(f"派生密钥: {key.hex()}")
```

## 面试要点

### Q1: 对称加密和非对称加密的区别？什么时候应该使用哪种？

**答:**

| 特性 | 对称加密 | 非对称加密 |
|------|----------|------------|
| 密钥 | 单密钥，加密解密相同 | 密钥对（公钥/私钥） |
| 速度 | 快（适合大数据） | 慢（适合小数据） |
| 用途 | 数据加密 | 密钥交换、数字签名 |
| 算法 | AES, ChaCha20 | RSA, ECC |
| 密钥分发 | 困难 | 容易（公钥可公开） |

**使用场景：**
- 对称加密：加密大量数据（文件、数据库、通信内容）
- 非对称加密：密钥交换、数字签名、小数据加密
- 实际应用：混合加密系统（非对称加密交换对称密钥，对称加密加密数据）

### Q2: 什么是前向保密 (Forward Secrecy)？为什么重要？

**答:** 前向保密是指即使长期私钥在未来被泄露，**过去的通信内容也不会被解密**。

**重要性：**
- 保护历史通信免受未来密钥泄露的影响
- 即使服务器私钥被盗，历史会话仍然安全

**实现方式：**
- 每次会话使用临时的 Diffie-Hellman 密钥交换
- 会话密钥不存储，会话结束后立即销毁
- TLS 1.3 强制要求前向保密

### Q3: 为什么密码不能直接用 SHA-256 存储？

**答:** SHA-256 等通用哈希算法设计目标是**速度快**，而密码存储需要**慢且有成本**的算法：

1. **速度快是劣势**：攻击者可以用 GPU/ASIC 快速尝试数十亿个密码
2. **缺乏盐值**：相同密码产生相同哈希，容易被彩虹表攻击
3. **没有内存困难**：容易被专用硬件并行攻击

**正确做法：**
- 使用 Argon2、bcrypt、scrypt 等密码哈希算法
- 内置盐值，每个密码使用不同的随机盐
- 可调节的成本因子（迭代次数、内存使用）

### Q4: 哈希函数和加密算法有什么区别？

**答:**

| 特性 | 哈希函数 | 加密算法 |
|------|----------|----------|
| 可逆性 | **单向**，不可逆 | **双向**，可以解密 |
| 输出 | 固定长度 | 与输入相关 |
| 密钥 | 不需要密钥 | 需要密钥 |
| 用途 | 完整性验证、指纹 | 数据保密 |
| 示例 | SHA-256, MD5 | AES, RSA |

**类比：**
- 哈希像指纹：从指纹无法还原整个人
- 加密像保险箱：用正确的钥匙可以打开取出内容

### Q5: 什么是数字签名？它如何保证消息的完整性和真实性？

**答:** 数字签名是发送方用**私钥**对消息哈希值进行加密的结果。

**验证过程：**
1. 接收方计算收到的消息的哈希值
2. 用发送方**公钥**解密签名，得到原始哈希值
3. 比较两个哈希值：如果相同，则消息未被篡改（完整性）
4. 只有持有私钥的人才能生成有效签名（真实性/不可否认性）

**使用的技术：**
- 哈希函数：确保任何消息改动都能被发现
- 非对称加密：私钥签名，公钥验证
