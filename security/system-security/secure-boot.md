## 概述

**安全启动 (Secure Boot)** 是 UEFI (Unified Extensible Firmware Interface) 的一项安全功能，确保系统在启动过程中只加载经过数字签名的可信软件。它是构建可信计算基础的关键组件，可有效防止恶意软件（如 bootkit、rootkit）在系统启动早期阶段植入。

├─────────────────────────────────────────────────────────────────┤

│   ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐     │
│   │   BIOS  │───▶│  MBR    │───▶│ Bootloader│──▶│   OS    │     │
│   │         │    │         │    │  (GRUB)   │    │         │     │

│        │              │               │              │         │

## 信任链 (Chain of Trust)

信任链是安全启动的核心概念，从硬件信任根开始，逐级验证软件组件。

```
┌─────────────────────────────────────────────────────────────────┐
│                   信任链架构                                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Root of Trust (信任根)                                        │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │  Hardware Root of Trust                                  │   │
│   │  ├─ CPU Microcode (微码完整性)                          │   │
│   │  ├─ UEFI Firmware (固件签名验证)                        │   │
│   │  └─ Platform Keys (平台密钥，存储在NVRAM)               │   │
│   │                                                          │   │
│   │  └─▶ Platform Key (PK) - 平台所有者密钥                 │   │
│   │      └─▶ Key Exchange Key (KEK) - 密钥交换密钥          │   │
│   │          ├─▶ db (Allowed Signatures) - 允许的数据库      │   │
│   │          └─▶ dbx (Forbidden Signatures) - 禁止的数据库   │   │
│   └─────────────────────────────────────────────────────────┘   │
│                              │                                  │
│                              ▼                                  │
│   Chain of Trust Verification                                   │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │  Step 1: UEFI Firmware 验证 Bootloader                  │   │
│   │          [PK/KEK] ──▶ db ──▶ 签名匹配? ──▶ 执行/拒绝   │   │
│   │                                                          │   │
│   │  Step 2: Bootloader 验证 OS Kernel                      │   │
│   │          [内置公钥] ──▶ 签名验证 ──▶ 加载内核           │   │
│   │                                                          │   │
│   │  Step 3: OS Kernel 验证 Drivers/Modules                 │   │
│   │          [内核密钥环] ──▶ 签名检查 ──▶ 加载模块         │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│   如果任何一步验证失败，启动将被阻止                               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## TPM (Trusted Platform Module)

TPM 是提供硬件级安全功能的专用芯片，是可信计算的硬件基础。

```
┌─────────────────────────────────────────────────────────────────┐
│                   TPM 架构与功能                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   TPM 核心功能:                                                   │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │  1. 安全密钥存储                                         │   │
│   │     • RSA/ECC 密钥对生成                                 │   │
│   │     • 私钥永不离开TPM                                    │   │
│   │     • SRK (Storage Root Key)                             │   │
│   │                                                          │   │
│   │  2. 平台完整性测量                                       │   │
│   │     • PCR (Platform Configuration Registers)            │   │
│   │     • 只增不减的测量日志                                 │   │
│   │                                                          │   │
│   │  3. 远程证明 (Remote Attestation)                        │   │
│   │     • 证明平台状态给远程验证方                           │   │
│   │     • 使用AIK (Attestation Identity Key) 签名            │   │
│   │                                                          │   │
│   │  4. 密封存储 (Sealing)                                   │   │
│   │     • 数据加密绑定到特定平台状态                         │   │
│   │     • 只有特定PCR值时才能解密                            │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│   TPM 版本对比:                                                   │
│   ┌──────────────┬─────────────────────┬─────────────────────┐  │
│   │   特性       │      TPM 1.2        │      TPM 2.0        │  │
│   ├──────────────┼─────────────────────┼─────────────────────┤  │
│   │ 算法支持     │ RSA, SHA1          │ RSA, ECC, SHA256等   │  │
│   │ 密钥层次     │ 固定3层             │ 灵活可配置           │  │
│   │ 授权机制     │ PCR, Locality      │ 增强授权策略         │  │
│   │ 标准规范     │ ISO/IEC 11889      │ ISO/IEC 11889        │  │
│   │ 当前状态     │ 已淘汰             │ 主流标准             │  │
│   └──────────────┴─────────────────────┴──────────────────────┘  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**TPM 使用示例：**

```bash
# 安装TPM工具
$ sudo apt install tpm2-tools

# 检查TPM是否可用
$ tpm2_getcap properties-fixed

# 查看PCR值
$ tpm2_pcrread

# 创建密钥
$ tpm2_createprimary -C o -g sha256 -G rsa -c primary.ctx

# 密封数据 (绑定到PCR)
$ echo "secret data" > secret.txt
$ tpm2_createpolicy --policy-pcr -l sha256:0,1,2,3 -L policy.pcr
```

## 面试要点

**Q1: Secure Boot和Measured Boot有什么区别？**
> Secure Boot通过验证数字签名阻止未授权代码执行；Measured Boot记录启动组件的哈希值到TPM的PCR中，用于远程证明和密封存储。两者常结合使用。

**Q2: 什么是信任链 (Chain of Trust)？**
> 信任链是从硬件信任根开始，逐级验证软件组件的机制。UEFI Secure Boot中：PK签名KEK，KEK签名db/dbx，db中的证书验证bootloader和OS签名。

**Q3: TPM中的PCR是什么？**
> PCR (Platform Configuration Register) 是TPM中用于存储测量值的寄存器。特性：1) 只增不减；2) 扩展操作是哈希链；3) 可用于密封存储，数据绑定到特定PCR值。
